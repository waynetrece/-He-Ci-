## 原因

HJ 提供的 `HJ網站客製需求表_1150422.pdf` 是一頁簡化架構表，不是完整流程規格。

如果我們先自己補很多流程，例如：

- 公私版自動拆單

會有兩個風險：

1. 系統被我們自己想得太複雜。
2. 客戶還沒確認現況，就先看到一堆進階功能，容易造成 scope 放大。

## 第一輪主軸

第一輪不要先問前台畫面細節，也不要先調整 mockup。

請先針對 HJ 需求表裡會影響架構與報價的核心系統提問：

1. **訂單管理**
   - 訂單什麼時候成立？
   - 是否需要後台幫客人修改訂單？
   - 出貨狀態有哪些？
   - 撿貨單 / 訂單匯出 / 退換貨第一版做到哪裡？

2. **金流**
   - 第一版是否只做需求表列的 4 類：一般匯款、綠界、宅配貨到付款、自取貨到付款？
   - 綠界要開哪些付款方式？
   - 匯款、貨到付款、退款由誰確認？

3. **物流與出貨**
   - 四大超商、宅配、自取是否都要做？
   - 物流商是客戶下單時選，還是 HJ 出貨時選？
   - 貨態是物流 API 回傳，還是 HJ 人工更新？

4. **ERP 串接**
   - 訂單匯入 ERP 是否一定要即時 API？
   - 顧客編號如何對應？
   - 庫存、商品、顧客、訂單哪些資料要即時同步？
   - 同步失敗時怎麼處理？

5. **庫存管理**
   - 庫存是否以 ERP 為準？
   - 何時檢查 / 預留 / 扣庫存？
   - 缺貨提醒要通知誰？
