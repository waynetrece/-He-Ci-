### 公版商品(題 2)

- 規格層級:尺寸、容量、顏色、材質、數量(箱、條)。
- 不同規格組合的價格分開、庫存分開、凌越各自編號。

### 樣品申請(題 1)

- 處理方式:直接成立樣品申請,照一般出貨流程(不需業務人工確認)。
- 申請門檻:必須登入會員。
- 適用範圍:全部公版商品。
- 數量限制:每品項 ≤ 3;最多 10 款品項;總數 ≤ 30。
- 收費規則:樣品免費,但收運費。
- 出貨規則:不可與販售公版商品同包出貨。
- **(2026-05-04 Wayne 確認)樣品運費 = 固定金額後台可調,不論幾件同價;樣品申請獨立成「樣品申請單」(後台與一般訂單分開列)。**

### 加購商品(題 3)

- 加購方式:購物車/結帳頁推薦 + 買特定商品才出現指定加購(兩種都要)。
- 加購品價格依會員等級不同。
- **(2026-05-04 Wayne 確認)加購品本質**:
  - 加購品本身就是一般公版商品(有自己的貨號、價格、庫存,可單買)。
  - 「加購」是定價優惠規則(原 $80 → 加購價 $50)。
  - 庫存扣加購品自身的庫存(沒有「加購品專用庫存」的概念)。
  - 同一個加購品可同時是「全站推薦」+「特定主商品搭配」,完全相容(HJ 答 1 & 3)。

### 私版商品(題 5)

- 私版的細節溝通、檔案傳遞、下單確認,全部走 LINE。
- 網站不做檔案上傳、線上校稿、線上確認完稿。
- 私版訂購流程:
  1. 私版商品頁選擇客製需求(HJ 後台勾選欄位機制)
  2. 成立私版詢價單
  3. 客戶加入官方 LINE,由客服溝通
  4. 雙方確認 → 客戶私下匯款訂金 → **業務在後台手動觸發「詢價單轉正式訂單」+ 同時進凌越**
- 私版商品頁設計「上傳檔案」按鈕,直接跳轉到 LINE 上傳。
- LINE 帶詢價單編號:技術前提齊備(OA + admin 權限),具體互動方式(LIFF / query string / 其他)先跳過,以後再決定。

### 公版 + 私版(題 20)

- **(2026-05-04 Wayne 確認)公版與私版兩條流程完全分開,不允許同一購物車送出。**
- 樣品也不能加進公版的購物車(走獨立樣品申請流程)。

## 最終流程

### 1. 公版商品購買

```text
商品頁 → 同頁切換規格選擇器 → 加入購物車
→ (可加購其他商品,加購價代替原價)
→ 結帳 → 一般訂單
```

### 2. 樣品申請(獨立流程)

```text
樣品入口 → 登入會員 → 選樣品(系統檢查 3/10/30 限制)
→ 選配送方式 → 計算樣品運費(固定金額)→ 付款
→ 樣品申請單成立(獨立於一般訂單)
```

### 3. 私版詢價

```text
私版商品頁 → 填客製需求(HJ 後台勾選的欄位)→ 送出詢價單(顯示詢價編號)
→ 引導加 LINE(理想上帶詢價單編號)→ HJ 客服在 LINE 確認價格與檔案
→ 客戶私下匯款訂金 → LINE 通知業務 → 業務在後台手動觸發「詢價單轉正式訂單」
→ 訂單進凌越 → 開始製作
```

### 4. 加購

```text
公版商品詳情/購物車/結帳推薦或指定主商品搭配
→ 客戶接受加購 → 加購品(自身公版)以加購價加入訂單
→ 扣加購品自身的庫存
```

### 5. 公版 / 私版前台分區

導覽分公版區與私版區。私版區可放「需要客製請從這邊」入口,但不混入公版商品頁的主流程。

```text
結帳 → 選一般匯款 → 訂單「待付款」(尚未進凌越) → 顯示匯款帳號
→ 客戶到 ATM/銀行匯款 → 回網站回填後五碼/截圖
→ HJ 會計核對 → 後台按「確認已付款」
→ 訂單「已付款」+ 進凌越 → 後台備貨
```

### 3. 宅配貨到付款(只限公版+加購)

```text
結帳 → 選貨到付款 → 訂單「待付款」+ 進凌越時點延後到收款後
→ HJ 出貨 → 物流商代收款 → HJ 收到代收
→ 業務後台切「已付款」+ 此時進凌越 → 訂單轉「已完成」
```

### 4. 自取現場付款

```text
結帳 → 選自取現場付款 → 訂單「待付款」(尚未進凌越)
→ HJ 備貨 → 通知客戶 → 客戶到場付款(現金/刷卡)
→ 業務後台切「已付款」+ 進凌越 → 訂單轉「已完成」
```

### 5. 自取線上信用卡

```text
結帳 → 選自取+線上信用卡 → 跳綠界刷卡 → 已付款 + 進凌越
→ HJ 備貨 → 通知客戶 → 客戶到場取貨 → 已完成
```

### 6. 退款

```text
退換貨情境 → 客戶 LINE/電話聯絡客服
→ 客服在後台選訂單「申請退款」
→ 路徑 A(信用卡):後台自動呼叫綠界退刷 API
→ 路徑 B(匯款):後台記錄,業務通知會計手動退
→ 訂單狀態切「已退款」(第 7 個狀態)
→ 凌越同步狀態
→ 處理發票(待 HJ 確認折讓單/作廢)
```
