# 進階功能備註（Wayne 內部口袋備案）

> 用途：HJ 第一輪簡報走「好推動但堪用」的版本，避免客戶看了想很多、覺得費用會越來越高。
> 本檔列出**主動拿掉、沒放進畫面 / Q 的進階功能**。HJ 在會議中如果主動提出，Wayne 可以拿這份出來討論「也可以這樣做」。
> 對應規格：本提案目前涵蓋公版商品 + 私版報價 + 會員系統 + 購物車閉環。
> 建立日期：2026-04-29

---

## 為何需要這份文件

第一輪示意給客戶的版本，原則是：
- **好推動**：不要讓客戶覺得功能複雜、費用會爆。
- **不漏功能**：核心交易閉環該有的都還在（購物車 / 結帳 / 付款 / 通知）。
- **進階功能不主動推**：避免客戶把「可以這樣做」當成「就要這樣做」。

但 HJ 內部有時候會主動問：「為什麼沒有 XX？」「我們其他系統有 XX，這套要嗎？」這時 Wayne 拿這份備註出來，能快速說明：「這是進階功能，可以加，但會推升費用 / 工期，建議先看核心需求。」

## 1 · 多門市分送（同一訂單分送多地址）

### 進階做法
一張訂單裡的不同商品可以指定不同收件門市。例如：
- 12oz 公版瓦楞紙杯 × 5,000 → 總店
- 牛皮紙便當盒（M）× 2,000 → 信義分店
- 客製禮盒 × 200 → 倉庫

並可選擇：
- **同單分送**：保持一張訂單，運費依各門市材積分別計算。
- **自動拆單**：結帳後系統拆成 N 張訂單，各自處理出貨 / ERP 開單 / 物流。

### 推動成本
- 結帳頁需要逐項收件人 UI、運費規則拆計、ERP 介接需支援同一訂單對多收件、物流單批次列印。
- 若採自動拆單，會員端要設計「子訂單」呈現，避免 1 筆下單→ N 筆通知造成困擾。

### 我們公司目前狀況
**沒做過這方式**（Wayne 4/29 確認）。要評估開發成本與 ERP 介接複雜度。

### 第一版採用方式
- 結帳頁只支援**單一收件地址**。
- 客戶要分送多門市，須分次下單。
- 如 HJ 要做，第二期評估。

### HJ 主動提時的話術
> 「這是進階功能，技術上可以做，但會牽動結帳 UI、運費試算、ERP 介接（同單對多收件）、物流批次。我們公司目前還沒做過。建議先確認 HJ 多門市客戶占比有多高、是否一次下單分送對 HJ 業務真的有效率，再決定要不要列入第一期。」

## 2 · 月結信用額度自動檢查

### 進階做法
- 系統維護每位月結客戶的**信用額度**（例如：合作 A 級 NT$ 200,000）。
- 訂單送出時系統自動檢查：當前未結金額 + 本單金額 ≤ 信用額度？
  - 是 → 訂單成立，自動扣額度。
  - 否 → 顯示「超過信用額度 NT$ X」，要求業務人工確認或請客戶改其他付款方式。
- 額度可由 ERP 同步、或在後台另設。
- 月結結帳後（出貨後 30 天）自動釋放額度。

### 推動成本
- 後台需新增「信用額度設定 / 維護」頁面。
- ERP 需提供額度查詢 API，或由網站獨立維護。
- 訂單成立流程要加入額度檢查邏輯（決定是否擋下單、改狀態、通知）。
- 額度不足的處理流程（業務確認 / 客戶改付款方式 / 部分扣抵）要設計清楚。

### 我們公司目前狀況
未做過這種額度即時扣 / 檢查邏輯。

### 第一版採用方式
- **所有月結訂單一律進「月結審核中」狀態，業務人工審核**。
- 不做額度自動檢查、不做額度自動扣 / 釋放。
- 優點：邏輯簡單、好推動、HJ 業務原本就需要對月結客戶有把關。
- 缺點：每筆月結都要人工審核，量大時會耗業務時間。

### HJ 主動提時的話術
> 「自動信用額度檢查可以做，但需要 HJ 先確認三件事：① 額度是 ERP 已有還是要新建？② 額度由誰維護（HJ 業務 / 會計 / 凌越）？③ 額度不足時要直接擋下單還是仍走人工審核？這三題定下來才能估開發。建議第一期先用人工審核版，量大再升級。」

## 3 · 公版 + 私版混單拆單規則

### 進階做法
購物車結帳後，系統依商品類型自動拆單：
- 公版商品 → 公版訂單（現貨即出，走標準物流）。
- 私版客製商品 → 私版訂單（製作期 2-4 週，製完才出，可能不同物流方式）。

或更細：依出貨日 / 倉庫 / 物流方式分組拆單。

### 推動成本
- 結帳流程與訂單編號規則要重新設計（一次結帳產生 N 張訂單編號）。
- 會員端要設計「群組訂單」UI，避免 1 筆結帳出現 N 筆通知。
- ERP 介接需支援同一結帳對多訂單，付款金額分攤要清楚。
- 退換貨需以子訂單為單位處理。

### 第一版採用方式
- **同一張訂單、分批出貨**。
- 公版部分先出，私版製完再出，但仍是同一個訂單編號。
- Q1 已詢問 HJ：是否要拆單？讓 HJ 決定。

### HJ 主動提時的話術
> 「公私版混單在出貨日 / 物流 / ERP 處理上的確會有麻煩。第一版先做『同單分批』，業務面比較單純。如果 HJ 業務發現混單造成困擾再升級成自動拆單。我們可以先看 HJ 目前公版客戶 vs 私版客戶有多少重疊，決定要不要列入第一期。」

## 4 · 其他可能被 HJ 提到的進階功能

下面列幾個常見的 B2B / 電商進階功能。如果 HJ 主動提，Wayne 可以延伸討論。

| 功能 | 第一版採用 | 進階版重點 |
|---|---|---|
| 滿額階梯運費 | 簡單規則：滿 NT$ 3,000 免運 | 依重量 / 材積 / 區域 / 物流商分別計算 |
| 訂單預訂出貨日 | 預估出貨日（系統算） | 客戶可指定出貨日 / 多次配送排程 |
| 重複下單 / 一鍵再訂 | 訂單詳情有「再訂一次」按鈕 | 自動定期重訂（每月 / 每週） |
| 報價有效期內保留庫存 | 不保留 | 報價成立後自動鎖定庫存 |
| 樣品申請扣庫存 | 不扣（樣品另計） | 與正式商品共用庫存、自動扣庫存 |
| 客戶自助退換貨 | 客戶在訂單詳情提出申請，業務審核 | 自動判定退換貨條件、自動退款 / 開立補寄單 |
| 多人共用後台帳號 | 一個會員一個帳號 | 企業帳號 + 多個子帳號（採購 / 會計 / 老闆） |
| 客製商品庫存追蹤 | 單純扣庫存 | 各客製選項組合獨立庫存（合併算 vs 分開算） |

## 使用方式

1. **不主動提**：Wayne 跟 HJ 簡報時，不主動講這些進階功能，避免客戶覺得「都要做」。
2. **HJ 主動問再拿出來**：例如 HJ 說「我們有些客戶想分送多門市」，Wayne 翻到 §1 對話。
3. **每項都有「第一版採用」+「推動成本」+「話術」**：方便 Wayne 即時回應。
4. **第一輪簡報時提供給 HJ 是 optional**：可以選擇給 / 不給。

### 建議：先不要把這份文件給 HJ
- 這份是 Wayne 內部備案，**邏輯是「我們先選了好推動的版本，但不漏功能」**。
- 給客戶會反而讓他們挑功能，不是聚焦在第一版要做的核心。
- 給 HJ 的版本應該只有 scope-checklist 客戶版（之後另整理），列他們要回答的問題。
