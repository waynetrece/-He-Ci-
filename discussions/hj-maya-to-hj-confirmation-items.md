# 瑪雅需再向 HJ 確認的項目

> 日期：2026-05-01
> 依據：窗口回覆文件 + 凌越 API 207 頁完整重讀
> 用途：這份只整理「HJ 必須決定的業務 / 流程 / 畫面需求」，不混入凌越 API 技術細節。

---

## 1. 最高優先：不確認會影響系統架構

### 1.1 樣品申請與一般商品的分流

目前已知：

- 全部公版商品都可申請樣品。
- 需登入會員。
- 每品項最多 3 個，最多 10 款，總數最多 30 個。
- 樣品免費但收運費。
- 樣品不可與販售公版商品一起出貨。

需要問 HJ：

1. 樣品運費是固定金額，還是依宅配 / 超商 / 材積 / 重量計算？
2. 客戶同時把樣品與一般商品加入購物車時，HJ 要哪一種流程？
   - 禁止一起結帳。
   - 同一結帳頁送出，但系統拆成「樣品申請」與「一般訂單」。
   - 可以一起付款，但後台拆單出貨。
3. 樣品是否需要扣庫存？
4. 樣品是否需要進凌越 ERP，還是只留網站後台管理？
5. 樣品是否需要會員中心查詢申請紀錄與出貨狀態？

若未確認的風險：

- 後續可能才發現要補拆單、樣品訂單、樣品運費、樣品出貨狀態與後台管理畫面。

### 1.2 私版詢價與 LINE 流程

目前已知：

- 私版細節、原始檔、確認下單都走 LINE。
- 網站不做檔案上傳、校稿、完稿確認。
- 網站成立私版詢價單後，引導客戶加 LINE。
- HJ 希望跳 LINE 時可帶詢價單編號。

需要問 HJ：

1. 私版詢價成立後，會員中心是否要能看到詢價紀錄與狀態？
2. 詢價單成立當下是否要進凌越 ERP，還是等 LINE 確認金額後才進？
3. 訂金支付後，是否才建立正式訂單？
4. LINE 帶詢價單編號的期望是哪一種？
   - 頁面顯示編號，客戶手動複製。
   - 點 LINE 後自動帶入預填文字。
   - 客服在 LINE 裡依客戶資訊手動查詢。
5. 私版詢價是否需要狀態，例如待客服回覆、報價中、待付訂金、已成立、已取消？

若未確認的風險：

- 後續可能漏掉詢價成功頁、詢價狀態、LINE 帶號、訂金付款、私版報價轉訂單等功能。

### 1.3 會員分級價與客戶價格規則

目前已知：

- 會員價格依地區與客戶類型，例如北部直客、中南部直客、北部盤商、中南部盤商。
- 會員等級由業務指定。
- 每個商品可設定不同等級價格。

需要問 HJ：

1. 實際會員價格類別總共有幾種？請列出完整清單。
2. 是否只會有固定幾種會員價，還是會有「單一客戶專屬價」？
3. 不同區域 / 客戶類型的價格是否可能超過 5 種？
4. 新會員未綁定 ERP 客戶編號前，要顯示一般價，還是需人工審核後才可下單？
5. 業務指定會員等級的流程是網站後台指定，還是凌越 ERP 指定後同步網站？

若未確認的風險：

- 若價格規則比 5 組價更複雜，商品與會員資料表會需要重做，前台價格也可能顯示錯誤。

### 1.4 訂單進凌越的時機

目前已知：

- HJ 希望網站下單後馬上出現在凌越。
- 狀態包含待付款、已付款、備貨中、已出貨、已完成、已取消。

需要問 HJ：

1. 未付款訂單是否也要立即進凌越？
2. 若凌越同步失敗，HJ 希望客戶端如何處理？
   - 直接不能完成下單。
   - 網站先成立訂單，後台標示同步失敗並人工補送。
   - 網站自動重送，超過次數再通知管理員。
3. 客戶是否可以自行取消未付款訂單？
4. 「已完成」7 天後自動跳轉，起算點是：
   - 已出貨日。
   - 物流送達日。
   - 管理員手動改為某狀態後第 7 天。
5. 訂單狀態是 HJ 人員在網站後台更新，還是以凌越狀態為準？

若未確認的風險：

- 後續可能需要補同步失敗頁、補送機制、訂單狀態映射與管理員處理流程。

## 2. 物流、材積與運費

### 2.1 材積與箱 / 條規則

目前已知：

- HJ 會填整箱與單條材積。
- 材積用於判斷超商配送與宅配運費。
- 超商限制為 45 x 30 x 30 cm、5 kg。
- Word 圖中已有宅配 / 超商、箱購 / 條購 / 箱購+條購規則。

需要問 HJ：

1. 「箱購」與「條購」在商品資料上如何判斷？是規格、單位，還是 SKU 類型？
2. 運費圖中的「才」是否就是 HJ 會填入的材積單位？
3. 多個商品混合時，材積如何加總？
4. 箱購 + 條購同時下單時，是否完全依 Word 圖規則計算？
5. 超商 45 x 30 x 30 cm 是看單一商品、單一箱、還是整張訂單包裹後尺寸？
6. 超商 5 kg 是看單一商品還是整張訂單總重量？
7. 超過超商限制時，前台是否只隱藏超商選項，或要顯示提示原因？

### 2.2 宅配、超商與自取

目前已知：

- 宅配公司由 HJ 出貨時填，不是客戶選。
- 宅配公司包含新航、嘉里大榮、新竹貨運、黑貓、大嘴鳥。
- 四大超商都要支援。
- 自取只有一個倉庫。
- 公版商品需填備貨時間。

需要問 HJ：

1. 宅配運送方式前台只顯示「宅配」，還是要讓客戶知道是哪一家物流？
2. 物流追蹤連結是由管理員貼網址，還是系統依物流公司 + 追蹤碼產生？
3. 超商寄件單目前 HJ 是用哪個平台列印？
4. 四大超商寄件單是否一定要在網站後台批次列印？
5. 自取是否需要「可取貨」狀態？
6. 自取通知客戶是由網站通知、LINE、電話，還是人工處理？

若未確認的風險：

- 若超商標籤列印被視為網站功能，但沒有指定物流服務商，開發時會卡住。

## 3. 商品資料、上架與匯入

目前已知：

- HJ 有 Shopline 商品 Excel。
- 商品資料包含編號、名稱、分類、規格、會員價、庫存、SEO 等。
- 需要商品批次匯入、錯誤報告、部分成功。
- 公版與私版都需要預約上架 / 下架。

需要問 HJ：

1. 商品資料正式來源是哪裡？
   - 凌越 ERP。
   - Shopline Excel。
   - 新網站後台。
   - 三者混合。
2. 商品圖片、SEO、商品描述、前台分類排序是否由網站後台管理？
3. 預約上架 / 下架是否只影響網站前台，不影響凌越商品停用？
4. 「提醒管理員並下架」的意思是：
   - 到時間自動下架並通知管理員。
   - 只提醒管理員，由人工下架。
   - 到時間前提醒，到時間自動下架。
5. 加購商品規則由誰維護？是否每個商品都可能設定不同加購品？
6. 私版商品是否也需要 SEO、預約上下架、分類與前台展示排序？

若未確認的風險：

- 後續可能補商品排程、SEO 欄位、加購規則、匯入錯誤頁與商品資料同步規則。

## 4. 會員與歷史訂單

目前已知：

- 新網站先建立會員，之後再補 ERP 客戶編號。
- 會員中心要看凌越歷史訂單。
- 再購買一次只限公版商品。
- 私版商品需重新報價。

需要問 HJ：

1. 舊客戶資料是否都有 Email、手機、統編，可以與新網站會員配對？
2. 若找不到 ERP 客戶編號，會員中心要如何顯示歷史訂單？
3. 歷史訂單要顯示哪些欄位？
   - 訂單日期。
   - 商品明細。
   - 金額。
   - 付款狀態。
   - 出貨狀態。
   - 發票。
   - 物流追蹤。
4. 舊訂單是否也要支援「再購買一次」？
5. 私版歷史訂單要顯示「重新詢價」按鈕嗎？

若未確認的風險：

- 後續會員中心可能漏掉 ERP 綁定、舊訂單查詢、再購買限制與私版重詢價流程。

## 5. 金流、退款與發票

目前已知：

- 綠界預計做信用卡。
- ATM 虛擬帳號還在考慮。
- HJ 尚無綠界帳號。
- 匯款需填匯款資料或上傳截圖。
- 信用卡退刷由綠界，匯款退款由會計。
- 客戶不能自行申請退貨，由 LINE / 電話聯繫客服，客服後台協助處理。

需要問 HJ：

1. 綠界帳號由誰申請？何時可提供測試與正式資料？
2. ATM 虛擬帳號是否確定要做？
3. 匯款截圖由誰審核？審核狀態是否要通知客戶？
4. 宅配貨到付款是否要進凌越訂單時標示特殊付款方式？
5. 發票由哪裡開立？
   - 凌越。
   - 綠界。
   - 另一套電子發票服務。
   - 人工開立。
6. 退貨 / 退款狀態是否要讓會員中心看得到？
7. 客服建立退貨時，是否需要選擇退貨商品、數量、退款方式、原因？

若未確認的風險：

- 後續可能漏掉匯款審核、金流 callback、退刷結果、發票載具、退款狀態與客服後台流程。

## 6. 庫存、預留庫存與缺貨提醒

目前已知：

- 商品庫存以凌越為準。
- HJ 希望凌越變動後網站馬上更新。
- 部分商品會預留給連鎖店，扣掉後才可在網站販售。
- 缺貨提醒通知業務或倉管。
- 缺貨門檻 HJ 可自行設定。
- HJ 詢問是否能轉到 LINE 提醒。

需要問 HJ：

1. HJ 能接受的庫存同步延遲是多少？秒級、分鐘級，還是只要結帳前準確？
2. 網站前台要顯示實際庫存數，還是只顯示有貨 / 缺貨？
3. 預留給連鎖店的數量在哪裡維護？
   - 凌越。
   - 網站後台。
   - 人工作業。
4. 預留庫存是否只影響前台可售量，不影響凌越總庫存？
5. 缺貨提醒要通知誰？
   - 業務。
   - 倉管。
   - 管理員群組。
   - 客戶。
6. LINE 提醒是要用官方帳號推播、群組通知，還是只產生待通知清單？

若未確認的風險：

- 後續可能才補庫存快取、結帳前二次確認、預留庫存表、LINE 通知設定與低庫存門檻管理。

## 7. 可先延後，但仍需排入規劃的項目

1. 訂單匯出用途：HJ 回覆仍是待討論，需確認是會計、出貨、對帳還是管理報表。
2. 撿貨單格式：按商品總量、按物流方式分組是否都要列印。
3. 後台批次匯入物流編號：Excel 欄位格式需確認。
4. 商品匯入錯誤報告格式：錯誤列、錯誤欄位、可下載報表。
5. 自取備貨時間：每商品設定後，訂單多商品時取最大天數還是逐項顯示。
