# 瑪雅需向凌越確認的 API / 文件補件項目

> 日期：2026-05-01
> 依據：凌越 5 頁 API 說明 + `最佳拍檔API.pdf` 207 頁完整重讀
> 用途：這份只整理「要跟凌越確認或請凌越補資料」的技術問題，方便瑪雅評估是否能直接製作。

---

## 1. 先給凌越的總結說法

目前文件已能確認凌越有 SOAP API，並有商品、客戶、訂單、報價、銷貨、銷退、收款、發票、庫存、倉庫、入出庫等資料結構。

但 HJ 專案要開發，還需要凌越提供「HJ 實際環境與實際範例」。目前 207 頁文件是通用欄位規格，不足以直接判斷每個流程能否無痛串接。

---

## 2. API 環境與權限

請凌越確認 / 提供：

1. HJ 測試環境 endpoint / WSDL。
2. HJ 正式環境 endpoint / WSDL。
3. 測試帳號、密碼、公司代號 `icpno`。
4. 正式帳號申請流程。
5. API 主機是否需固定 IP 白名單。
6. 是否支援 HTTPS / TLS，以及憑證要求。
7. `LyGetPassKey` 金鑰 30 秒失效，凌越建議每次呼叫前都重新取 key，還是可短暫 cache？
8. API 是否有呼叫頻率限制、單次筆數限制、分頁建議筆數。
9. HJ 帳號被授權可用哪些 `idakd`。
10. 不同 `idakd` 是否需分別開權限，例如查詢、轉入、刪除。
11. 錯誤碼完整清單與可重送 / 不可重送判斷。

---

## 3. 請凌越確認 HJ 專案需要啟用的資料類別

### 3.1 第一階段核心

請確認 HJ 是否可使用以下資料類別，並提供各自範例 XML：

| idakd | 名稱 | 用途 |
|---|---|---|
| `000000` | 商品基本資料 | 商品 / SKU / 規格 / 價格 / 材積 / 重量 |
| `00000B` | 單位換算設定 | 箱 / 條 / 輔助單位 |
| `00000G` | 單位設定 | 數量、重量、材積、付款方式、運送方式代碼 |
| `000001` | 客戶基本資料 | 會員與 ERP 客戶綁定 |
| `0000A0` | 訂單 | 網站公版訂單轉入 ERP |
| `0000A5` | 客戶報價單 | 私版詢價單可能對應 |
| `000009` | 目前庫存 | 前台庫存與結帳前可售量檢查 |
| `0000A1` | 銷貨單 | 歷史交易 / 出貨後資料 |
| `0000A2` | 銷貨退回單 | 退貨 / 銷退 |
| `0000A3` | 收款單 | 付款完成後同步收款資料 |
| `0000AS` | 銷項發票 | 發票資料 |
| `000004` | 倉庫 | 倉庫 / 可售庫存範圍 |
| `000006` | 貨品類別 | 商品分類 |
| `000007` | 客戶類別 | 會員 / 客戶類型 |
| `0000B7` | 出庫單 | 出貨 / 出庫資料 |

### 3.2 文件清單有提到，但 207 頁需補完整欄位或流程的項目

1. 客戶貨品項號對照或客戶別商品價格表：請提供完整 API 代碼、欄位、範例。
2. 撿貨單：文件清單若有列出，請提供完整 API 代碼、欄位、範例。
3. 物流公司 / 物流編號相關欄位：若不在上述單據內，請提供對應文件。
4. 超商寄件單 / 批次列印：若凌越有支援，請提供 API 文件；若沒有，請明確回覆需另接物流服務。

## 4. 商品與價格

請凌越確認：

1. 商品唯一識別是否以 `SK_NO` 為準。
2. HJ 每個規格組合都有 ERP 商品編號，是否表示每個規格組合都會是一筆 `000000` 商品？
3. 箱 / 條單位建議使用哪些欄位？
   - `SK_UNIT`
   - `SK_AUX_Q`
   - `SK_AUX_U`
   - `00000B` 單位換算
4. 整箱材積、單條材積、重量、體積建議對應哪些欄位？
5. 商品圖片欄位 `SK_FIGN~SK_FIGN4` 是檔名還是可回傳 URL？
6. 圖片實體檔如何取得？是否只能讀 `C:\inetpub\wwwroot\ERPAPI\OUTIMG\`？
7. 商品 SEO、商品描述、網站上架 / 下架日期是否有凌越欄位？若沒有，是否建議網站自管？
8. `SK_PRICE1~5` 與 `SK_FPRICE1~5` 如何對應 HJ 會員價？
9. 若 HJ 價格類別超過 5 種，凌越是否有客戶別商品價格或專案價 API？
10. 商品停用 `SK_STOP` 是否代表網站也要下架？還是只代表 ERP 停用？

需要範例：

- 一筆含箱 / 條單位的商品 XML。
- 一筆含 5 組價格與輔助單位價格的商品 XML。
- 一筆含圖片、重量、材積的商品 XML。

6. 訂單運費應作為：

   - 加項金額。

   - 其他欄位。
7. 折扣、加購品、樣品、贈品應如何放入明細？

## 13. 初步判定哪些可直接做，哪些需補資料

| 項目 | 目前判定 |
|---|---|
| 商品基本資料串接 | 可評估，但需補圖片、SEO、上架欄位邊界 |
| 客戶資料串接 | 可評估，但需補查詢條件與 HJ 實際客戶欄位 |
| 公版訂單轉入 | 可評估，需補 XML 範例、單號、未付款、失敗處理 |
| 私版詢價轉報價 | 理論可行，需 HJ 決定流程並請凌越補報價範例 |
| 庫存查詢 | 可評估，需確認可用量、預留庫存、同步頻率 |
| 會員歷史訂單 | 可評估，需決定讀訂單或銷貨單 |
| 收款同步 | 可評估，需補與訂單關聯方式 |
| 退貨 / 銷退 | 可評估，需補退款與發票作廢流程 |
| 電子發票 | 欄位看起來足夠，但需確認實際開立責任與範例 |
| 超商標籤列印 | 目前文件不足，需凌越明確回覆是否支援 |
| LINE 通知 | 不屬於凌越 API，應由網站 / LINE OA 評估 |
| 綠界金流 | 不屬於凌越 API，應由網站 / 綠界評估 |
