> 用途：先依 HJ 原始需求表的規劃提問，確認訂單、金流、物流、ERP、庫存等核心系統邊界；取得回覆後，再評估 mockup 與功能建議。

## 核心原則

1. **先照 HJ 的規劃問，不先替客戶擴充**
   - PDF 是簡化需求架構，不是完整流程規格。
   - 第一輪目標是確認 HJ 原本想做的範圍，不是設計新功能。

2. **先問核心系統，再談前台畫面**
   - 訂單、金流、物流、ERP、庫存會決定整個系統怎麼做。
   - 前台 mockup 要等這些答案回來後再收斂。

3. **先問現況，再提建議**
   - 先問 HJ 現在怎麼做。
   - 再問網站第一版是否照現況。
   - 拿到答案後，才判斷是否需要提出更好的流程。

4. **未寫在需求表的進階功能先不主動問**
   - 月結 30 天、多門市分送、自動信用額度、自動拆單等，先放內部備案。
   - 除非 HJ 主動提，否則不放進第一輪客戶問題。

## 第一輪：核心系統必問

### 1. 訂單管理

需求表寫：
- API 串接：訂單匯入 ERP。
- 批次列印 / 產生物流單。
- 訂單出貨狀態。
- 撿貨單匯出：統整全部要出貨商品的數量。
- 訂單資料匯出。
- 可幫客人修改訂單。
- 退換貨。

要問 HJ：
- 訂單在網站送出後，什麼時候算正式成立？送出就成立、付款完成才成立，還是 HJ 人工確認後才成立？
- HJ 是否需要在後台幫客人修改訂單？通常會改哪些內容：數量、收件資料、付款方式、商品、取消品項？
- 訂單出貨狀態目前有哪些名稱？例如：待確認、已成立、備貨中、已出貨、已完成、已取消。
- 撿貨單要以什麼方式匯出？按日期、訂單、商品彙總，還是物流批次？
- 退換貨目前怎麼處理？網站第一版只需要提供申請入口，還是要做完整退換貨流程？

第一版先確認：
- 訂單狀態名稱與順序。
- 訂單修改由誰做。
- 退換貨先做到哪裡。
- 撿貨單 / 訂單匯出是否第一版必做。

### 2. 金流

需求表寫：
- 一般匯款。
- 綠界。
- 宅配貨到付款。
- 自取貨到付款。

要問 HJ：
- 第一版是否只做需求表這 4 類金流？
- 綠界要開哪些付款方式：信用卡、ATM、超商代碼，還是只開信用卡？
- 一般匯款付款後，誰確認入帳？網站是否需要上傳匯款資訊？
- 貨到付款是否只適用公版商品？私版客製商品是否也可用貨到付款？
- 退款或取消訂單時，金流後續由誰處理？

第一版先確認：
- 付款方式清單。
- 每種付款方式對應的訂單狀態。
- 付款失敗 / 未付款 / 取消訂單的處理方式。

### 3. 物流與出貨

需求表寫：
- 四大超商。
- 多家宅配：出貨時可選擇物流公司，以利顧客查詢貨態。
- 自取。
- 批次列印 / 產生物流單。
- 訂單出貨狀態。

要問 HJ：
- 四大超商、宅配、自取是否第一版都要做？
- 多家宅配是客戶下單時選，還是 HJ 出貨時才選？
- 物流單是否要從網站批次產生？
- 物流狀態是由物流商 API 回傳，還是 HJ 後台人工更新？
- 自取訂單是否也要有付款與取貨狀態？
- 離島地區是否一律改由客服確認，不走一般配送？

第一版先確認：
- 客戶能選哪些配送方式。
- HJ 後台能做哪些出貨操作。
- 會員端要顯示哪些物流資訊。

### 4. ERP 串接

需求表寫：
- 訂單匯入 ERP。
- 庫存即時更新。
- 網站客人需與原 ERP 客戶編號相同。

要問 HJ：
- 凌越 ERP 是否已有 API 可供新網站串接？
- 訂單、顧客、庫存、商品資料，哪些一定要即時同步？
- 若第一版先半自動或人工匯入，HJ 是否可接受？
- ERP 匯入失敗時，網站訂單是否仍成立，還是要擋下單？
- 新網站會員如何對應原 ERP 客戶編號？

第一版先確認：
- 哪些資料由 ERP 當主資料來源。
- 哪些資料由網站產生後送 ERP。
- 同步失敗時由誰處理。

### 5. 庫存管理

需求表寫：
- API 串接：庫存即時更新。
- 部分商品需要預留庫存。
- 缺貨提醒。

要問 HJ：
- 庫存數量是以 ERP 為準，還是網站也會維護庫存？
- 庫存要在商品頁、加入購物車、結帳送出哪個階段檢查？
- 「預留庫存」適用哪些商品？加入購物車就預留，還是訂單成立後才預留？
- 缺貨提醒要提醒誰：客戶、HJ 業務、倉管，還是網站後台顯示？
- 私版商品是否需要庫存概念，還是只做報價 / 接單？

第一版先確認：
- 庫存來源。
- 庫存扣除與預留時機。
- 缺貨時前台要怎麼顯示。

## 第二輪：交易相關但可等核心答案後問

### 6. 公版商品下單

需求表寫：
- 公版商品系列：提供給客人下單。
- 加購商品。
- 離島地區需與客服聯絡下單。

要問 HJ：
- 公版商品是否都能直接下單？
- 哪些商品因運費、材積、庫存或地區限制需要客服確認？
- 加購商品是商品頁加購，還是購物車推薦？

### 7. 私版商品報價與 LINE

需求表寫：
- 客人在網站上點選需求選項得到報價。
- 部分複雜客製商品，轉 LINE 客服報價。
- 客服、客人提供製作原始檔、下單，均轉 LINE 客服。

要問 HJ：
- 哪些私版商品可以網站即時報價？
- 哪些一定要轉 LINE 客服？
- 原始檔是否全部走 LINE，不做網站上傳？
- 私版報價完成後，是網站結帳，還是仍由 LINE 客服確認下單？

### 8. 公版與私版是否共用訂單流程

需求表把公版商品系列與私版商品系列分開列，沒有明寫混在同一張訂單。

要問 HJ：
- 公版商品與私版客製商品目前是否分開處理？
- 若客戶同時買公版與私版，目前 HJ 是分開開單還是合併處理？
- 網站第一版是否照 HJ 現況：公版走購物車，私版走報價 / LINE 確認？

備註：
- 不先問自動拆單、多門市分送、複雜分批出貨。
- 等 HJ 回覆現況後，再決定 mockup 要不要拆開。

### 9. 會員與顧客分級

需求表寫：
- 註冊方式：LINE、Email（需驗證）。
- 查詢歷史訂單，可再購買一次按鈕。
- 訂單配送狀態。
- 網站客人需與原 ERP 客戶編號相同。
- 多層會員分級：商品會因顧客分級而有不同價。

要問 HJ：
- LINE 與 Email 是否都要做註冊 / 登入？
- 會員分級有幾級，價格是否由 ERP 決定？
- 歷史訂單只看新網站訂單，還是要匯入 ERP 舊訂單？
- 「再購買一次」是否只針對公版商品？

需求表寫：
- 公版商品：Excel 批次匯出匯入、預約上架、商品頁影片或 3D 圖。
- 私版商品：Excel 批次匯出匯入、預約上架。

要問 HJ：

- 預約上架是否第一版必做？
- 商品影片 / 3D 圖是每個商品都要支援，還是少數商品才有？

### 11. 樣品申請

需求表寫：

要問 HJ：
- 哪些商品可以申請樣品？

- 樣品申請後由誰審核？
- 樣品是否需要進訂單 / ERP / 出貨流程？

### 12. 網頁內容與活動公告

需求表寫：
- 首頁、關於我們、訂購流程、購物須知。
- 部落格、活動公告等，需可設定發佈日期。

要問 HJ：
- 哪些內容需要 HJ 後台自行編輯？
- 活動公告是否需要預約發佈與下架？
- 彈跳視窗 / 浮動視窗是否第一版必做？
- 公版與私版商品頁底色是否已有設計規範？

### 13. 數據分析與行銷串接

需求表寫：
- 商品瀏覽。
- 商品加入購物車。
- 商品購買量。
- Google Ads、Meta、SEO。

要問 HJ：
- 三個數據指標要在網站後台看，還是接 GA4 / GTM？
- Google Ads / Meta 是只埋追蹤碼，還是要做事件追蹤？
- SEO 範圍是基本 meta / sitemap，還是需要文章與關鍵字策略？

### 14. 網站維護與權限

需求表寫：
- 防火牆保護。
- SSL 憑證。
- 網站保固。
- 系統諮詢。
- 登入紀錄、修改記錄。
- 管理人員功能權限設定。

要問 HJ：
- 正式網域與 DNS 由誰管理？
- SSL 與資安防護由誰設定與續約？
- 網站保固期與範圍怎麼定義？
- 後台需要哪些角色：老闆、業務、客服、倉管、會計？
- 各角色可以看 / 新增 / 修改 / 匯出哪些資料？

## 給 Wayne 的第一輪建議

第一輪先問 5 組核心系統，不要先問畫面細節：

1. **訂單管理**
   - 訂單何時成立？
   - 是否需要後台改訂單？
   - 出貨狀態、撿貨單、退換貨第一版要做到哪裡？

2. **金流**
   - 第一版是否只做需求表 4 種金流？
   - 綠界要開哪些付款方式？
   - 匯款 / 貨到付款 / 退款由誰確認？

3. **物流**
   - 四大超商、宅配、自取是否都要做？
   - 物流商由客戶選還是 HJ 出貨時選？
   - 貨態是 API 回傳還是人工更新？

4. **ERP**
   - 訂單匯入 ERP 是否一定要即時 API？
   - 顧客編號如何對應？
   - 同步失敗時怎麼處理？

5. **庫存**
   - 庫存是否以 ERP 為準？
   - 何時檢查 / 預留 / 扣庫存？
   - 缺貨提醒要通知誰？

若窗口時間很短，先問最關鍵 3 題：

1. 訂單何時成立，是否一定要即時匯入 ERP？
2. 金流與物流第一版要做到需求表哪幾項？
3. 庫存是否以 ERP 為準，是否要即時同步？
