## 白話化原則（Wayne 4/29 晚指示）

1. 每題只問一件事，不要一題塞太多判斷。
2. 技術詞要換成白話，或旁邊立刻補一句說明。
   - API → 系統自動串接
   - 排程批次 → 固定時間匯出 / 匯入一次
   - ERP 匯入失敗 → 訂單資料送不進凌越
   - 預留庫存 → 先把庫存扣住，避免別人買走
3. 問法先確認「HJ 現在怎麼做」，再問「網站初期要不要照這樣做」。
4. A/B/C 選項要像窗口可以直接選，不要像技術規格。
5. 不要讓問題看起來像我們在推功能。月結、自動拆單、自動同步、物流 API 保持中性。
6. 31 題作為內部總表，實際 LINE 分批送，每批要短、好回。

優先重寫的題目：ERP 串接、庫存預留、物流狀態自動更新、退款 / 取消訂單分工、公版 + 私版混單。

## 整合版 LINE 訊息母稿（請依上面分批送，不要一次整包貼）

```
嗨～想跟你確認 HJ 一些業務面的細節，幫我評估網站要做到什麼程度。每題都很簡單，可以分批回，先了解你們現在怎麼做就好。


【1. 公版 + 私版訂單什麼時候算成立】

1.1 客戶在網站送出公版商品訂單後，HJ 這邊什麼時候算這張訂單已經成立、可以開始備貨？
   (A) 客戶按送出就算
   (B) 客戶付完款才算
   (C) 業務看過、確認過才算

1.2 客戶確認私版客製報價、送出訂單後，什麼時候算這張訂單已經成立、可以開始安排製作？
   (A) 客戶確認報價就算
   (B) 客戶付完款才算
   (C) 業務再確認規格 / 條件，OK 後才算

1.3 如果客戶把現貨可即出的公版商品，跟需要製作的私版客製品，一起放購物車結帳，HJ 希望網站怎麼處理？
   (A) 一張訂單就好，公版先出貨，私版做好再補出
   (B) 分成兩張訂單處理（由 HJ 決定要人工分，或之後再評估系統處理）
   (C) 規定客戶要分兩次結帳（公版一筆、私版一筆）
   (D) HJ 沒遇過 / 不確定


【2. 訂單管理】

2.1 HJ 業務有時候需要在後台幫客戶改訂單嗎？通常會幫他們改哪些東西？
   常見有：數量、收件人地址、付款方式、加減商品、取消整張或部分品項。
   想知道 HJ 的習慣，網站後台需不需要做這個功能。

2.2 一張訂單從進來到結束，HJ 內部會經過哪些狀態？
   例如：待確認、已成立、備貨中、已出貨、已完成、已取消。
   HJ 用的名稱是什麼？或還有其他狀態？

2.3 「撿貨單」就是倉庫員工早上拿這張單子，照單去架上把當天要出貨的東西一次拿齊。HJ 目前是怎麼整理出貨清單給倉管的？
   (A) 把當天所有訂單的同一商品加總（例：今天總共要出 200 個紙杯）
   (B) 一張訂單列一張單，倉管照單拿
   (C) 按物流商分批（黑貓的訂單一張、超商的一張）
   (D) 還沒做，業務 / 倉管自己找
   (E) 其他作法

2.4 退換貨目前是怎麼處理的？（客戶打電話、LINE@、Email？）
   想確認網站初期：
   (A) 只做「客戶在網站填申請單」這個入口，後面流程還是 HJ 業務手動處理
   (B) 整個退換貨流程都做進網站（申請 → 審核 → 退款 → 補寄）
   (C) 不需要，沿用現有方式


【3. 金流（付款方式）】

3.1 需求表上寫了 4 種付款方式：一般匯款、綠界、宅配貨到付款、自取貨到付款。網站初期就做這 4 種嗎？要不要增減？

3.2 如果用綠界（最常用的金流服務商），HJ 想開放客戶用哪幾種？
   - 信用卡
   - ATM 轉帳（綠界給虛擬帳號）
   - 超商代碼繳費
   - 全部都開
   - 還沒決定

3.3 客戶選「一般匯款」後，HJ 那邊現在是怎麼確認他匯款了？
   (A) 客戶要傳匯款帳號末 5 碼 / 截圖給業務
   (B) 業務每天看銀行帳戶對帳
   (C) 其他作法
   想知道網站需不需要做「上傳匯款資料 / 末 5 碼填寫」這個功能。

3.4 貨到付款，HJ 通常只給公版商品用嗎？私版客製品（已經做好的東西）也可以貨到付款嗎？
   說明：私版客製通常金額大，怕客戶拒收，業界常會限制不能用貨到付款。HJ 怎麼想？

3.5 (補充題) HJ 業務上有沒有讓客戶「先出貨、月底一起結帳」這種月結客戶？
   說明：需求表沒列月結，這題只是補確認。如果 HJ 沒有做月結客戶，直接回「沒有」就好。
   有的話想了解：
   - 哪些客戶可以月結？業務看交情，還是有級別？
   - 月結訂單，HJ 業務通常會先看一下再放行嗎？還是過去往來都正常的就直接出？
   - 由業務 / 主管 / 會計誰看？

3.6 客戶要退款或取消訂單時，HJ 內部通常會做這幾件事，每件可能不同人負責，想了解 HJ 目前的分工：
   (a) 把錢退給客戶（信用卡退刷、退回他的銀行帳戶；若有月結，再看是否扣抵）— 由誰處理？
   (b) 發票要作廢或重新開折讓單 — 由誰處理？
   (c) 商品庫存要把扣掉的加回來 — 由誰處理？
   (d) 凌越裡的訂單也要對應改掉 — 由誰處理？


【4. 物流與出貨】

4.1 客戶結帳的時候，HJ 想開放他們選哪幾種收貨方式？
   - 四大超商（7-11、全家、萊爾富、OK）
   - 宅配（黑貓、新竹貨運、嘉里大榮等）
   - 自取（去 HJ 倉庫 / 公司拿）
   網站初期是全部都要，還是先做其中幾種？

4.2 「物流單」就是出貨那天貼在箱子上、給物流司機掃的那張託運單。HJ 目前是怎麼產生這張單的？
   (A) HJ 把訂單資料傳給物流商系統，物流商系統印出來
   (B) HJ 自己用 Excel 或內部系統手動填
   (C) 還沒做 / 司機來收件時手寫
   想知道網站初期是不是要做「批次列印物流單」功能。

4.3 客戶想在網站上看自己的訂單「已出貨 / 配送中 / 已送達」，HJ 希望怎麼做到？
   (A) 跟物流商系統連在一起，狀態自動更新到網站（功能比較貴，但客戶不用問就知道）
   (B) HJ 出貨後在後台自己改狀態（成本低，但要人手動點）
   (C) 不做這個，網站只放物流追蹤連結，客戶自己點去物流商查
   說明：A 需要跟物流商另外做系統連接，會增加開發費用；B 成本低但要有人手動更新。

4.4 客戶選「自取」時，HJ 目前通常怎麼處理付款跟取貨？
   想確認網站要不要照這個方式顯示付款狀態（已付款 / 待付款）跟取貨狀態（待取 / 已取）：
   (A) 客戶到現場才付款 + 取貨
   (B) 客戶先在網站付款，再來取
   (C) 兩種都要支援

4.5 離島地區（澎湖、金門、馬祖等）的訂單，HJ 通常要另外溝通運費跟配送方式，因為跟本島不一樣。網站要：
   (A) 直接擋下，必須走客服 / LINE 聯絡
   (B) 一樣讓他們下單，HJ 接到單再處理
   (C) 列在運費表裡，自動算好

4.6 網站的運費要怎麼算？
   (A) 統一一個金額（例：一律 NT$ 150）
   (B) 依重量算
   (C) 依材積（紙箱大小）算
   (D) 依縣市區域算
   (E) 滿多少免運（例：滿 NT$ 3,000 免運）
   (F) 其他規則 / 混合
   合作客戶（A 級 / B 級會員）有特別運費優惠嗎？


【5. ERP 串接（凌越）】

5.1 客戶在網站下完訂單之後，這張訂單的資料會不會自動出現在凌越裡？還是業務要手動處理？
   (A) 自動而且很快（客戶下完單，凌越馬上就有這筆訂單）
   (B) 自動但有時間差（每 N 分鐘 / 每天固定時間，網站訂單會被匯入凌越一次）
   (C) 不會自動，業務每天要手動把網站訂單匯出再丟進凌越
   (D) 還沒做過，或目前只有舊網站 / 舊系統有處理

5.2 網站跟凌越可能會有 4 種資料需要互相更新。HJ 希望哪些要馬上更新？哪些一天更新一次也可以？哪些人工整理就好？
   (a) 客戶在網站下訂單 → 進凌越
   (b) 客戶在網站註冊 / 改資料 → 凌越也要更新
   (c) 凌越改商品庫存 → 反映到網站（避免賣到沒貨的）
   (d) 凌越改商品價格 / 上架新品 → 反映到網站
   每一項都可以選：
   - 馬上更新（一邊改，另一邊馬上知道）
   - 一天一次或固定時段更新
   - 業務自己手動匯出匯入

5.3 萬一網站要把訂單資料送給凌越，但沒送進去（網路斷、凌越在維護、欄位不符等），這時候 HJ 希望怎麼辦？
   (A) 訂單還是讓客戶成立，凌越那邊事後補；告訴 HJ 業務 / 系統管理員去處理
   (B) 擋下這張訂單，請客戶稍後再試（避免業務漏單）
   (C) 還沒想過，沿用現在 SHOPLINE 的做法

5.4 凌越裡每個客戶有自己的編號（例：C001、C002）。如果客戶用同一個 Email 在新網站註冊，HJ 希望網站怎麼跟凌越的客戶資料對在一起？
   (A) 老客戶用 Email 比對，自動對到凌越客戶編號
   (B) 業務在後台手動把網站會員 vs 凌越客戶編號做配對
   (C) 新網站會員另外給新編號，跟凌越的不對（兩邊各自獨立）
   (D) 還沒想過

5.5 網站剛上線時，如果暫時做不到「所有資料都自動在網站和凌越之間更新」，HJ 可以接受先用「業務每天手動匯出匯入一次」當過渡嗎？還是必須一次到位？
   (A) 可以先過渡，之後再升級成自動
   (B) 必須一次到位，凌越那邊我們會去想辦法
   (C) 看時間 / 預算決定


【6. 庫存】

6.1 商品的庫存數量，HJ 希望以哪邊為準？
   (A) 凌越為主：網站只是顯示，凌越庫存改了網站就跟著改
   (B) 網站自己也維護一份：例如預留 100 個給網站賣，賣完再從凌越調
   (C) 兩邊各自，網站賣公版、凌越管整體
   (D) 還沒想過

6.2 HJ 現在接單時，通常在什麼時間點確認還有沒有貨？網站之後要不要也照這個時間點檢查？
   (A) 商品頁就顯示「剩 X 個」/「已售完」
   (B) 客戶把商品加入購物車時才檢查
   (C) 客戶結帳送出時才檢查
   (D) 以上都要
   說明：越早檢查，客戶越不容易買到沒貨的商品；但也需要跟凌越配合得更緊。

6.3 「預留庫存」就是先把客戶可能買的數量扣下來、不讓別人買走。HJ 哪些商品需要這樣做？
   (A) 高單價 / 客製私版要預留（避免客戶下單時別人剛好買走）
   (B) 公版常規商品也要預留
   (C) 全部都要預留
   (D) 不需要預留，先到先得就好
   如果需要預留，再確認什麼時候預留：
   - 加入購物車就先扣（客戶 30 分鐘內沒結帳就釋放）
   - 訂單成立才扣

6.4 商品快沒貨 / 沒貨時，HJ 現在通常由誰知道、誰處理？網站之後要不要通知這些人？(可複選)
   - 客戶（網站顯示「缺貨可訂閱通知」，到貨自動寄信）
   - HJ 業務（提醒可以聯絡客戶 / 補貨）
   - 倉管（提醒要進貨）
   - 不通知，後台顯示給管理員自己看

6.5 私版客製商品（每次客戶要的規格 / 圖案都不一樣）需要算「庫存」嗎？還是純粹接單後再做？
   (A) 不算庫存，純接單後排製造
   (B) 還是要算原料庫存（例如紙、印刷油墨）
   (C) 部分有半成品庫存（例如已開模的標籤）


【7. 發票】

7.1 企業客戶現在下單時，會怎麼提供「公司抬頭 + 統編 + 發票寄送地址」？網站之後要怎麼處理？
   (A) 客戶第一次填，之後系統記住，下單時自動帶（客戶可改）
   (B) 每次都重新填一次（避免帶錯）
```
