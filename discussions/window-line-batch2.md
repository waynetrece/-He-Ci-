## LINE 訊息（直接複製貼）

```
嗨～接續上次的 6 題，再跟 HJ 確認 5 個核心系統目前的做法。可以分批回，重點是先了解你們現在怎麼做，再來談第一版做到哪裡。


【1. 訂單管理】

- 訂單在網站送出後，什麼時候算正式成立？(送出就成立 / 付款完成才成立 / HJ 人工確認後才成立)
- HJ 是否需要在後台幫客人修改訂單？通常會改哪些內容（數量 / 收件 / 付款方式 / 商品 / 取消品項）？
- 訂單出貨狀態目前有哪些名稱？(例：待確認、已成立、備貨中、已出貨、已完成、已取消)
- 撿貨單要以什麼方式匯出？(按日期 / 訂單 / 商品彙總 / 物流批次)
- 退換貨目前怎麼處理？網站第一版只需要申請入口，還是要做完整退換貨流程？


【2. 金流】（跟前次「付款方式」可一起回）

- 第一版是否只做需求表這 4 類：一般匯款、綠界、宅配貨到付款、自取貨到付款？
- 綠界要開哪些付款方式：信用卡 / ATM / 超商代碼？或只開信用卡？
- 一般匯款付款後誰確認入帳？網站是否需要客戶上傳匯款資訊？
- 貨到付款只適用公版商品嗎？私版客製商品也可用嗎？
- 退款 / 取消訂單時，金流後續由誰處理？


【3. 物流與出貨】（運費前次已問，這份補出貨流程）

- 四大超商、宅配、自取，第一版是否都要做？
- 多家宅配是「客戶下單時選」，還是「HJ 出貨時才選」？
- 物流單要從網站批次產生嗎？
- 物流狀態是物流商 API 回傳，還是 HJ 後台人工更新？
- 自取訂單是否也要有付款與取貨狀態？
- 離島地區是否一律改由客服確認？


【4. ERP 串接】（同步機制前次已問，這份補資料邊界）

- 凌越 ERP 是否已有 API 可供新網站串接？
- 訂單、顧客、庫存、商品資料，哪些一定要即時同步？哪些半自動或人工匯入即可？
- ERP 匯入失敗時，網站訂單是否仍成立還是擋下單？
- 新網站會員如何對應原 ERP 客戶編號？
- 第一版若先用半自動或人工匯入，HJ 是否可接受？


【5. 庫存管理】

- 庫存數量是以 ERP 為準，還是網站也會自己維護？
- 庫存要在哪個階段檢查？(商品頁 / 加入購物車 / 結帳送出)
- 「預留庫存」適用哪些商品？加入購物車就預留，還是訂單成立後才預留？
- 缺貨提醒要提醒誰：客戶、HJ 業務、倉管，還是只在後台顯示？
- 私版商品需要庫存概念嗎？還是只做報價 / 接單？

---

時間很短的話，最關鍵的 3 題：
1. 訂單何時成立？是否一定要即時匯入 ERP？
2. 金流與物流第一版要做到需求表哪幾項？
3. 庫存是否以 ERP 為準？是否要即時同步？
```
