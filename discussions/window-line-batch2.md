## 整合版 LINE 訊息全文（直接複製貼）

```
嗨～想跟你確認 HJ 一些業務面的細節，幫我評估網站要做到什麼程度。可以分批回，重點是先了解你們現在怎麼做，再來討論網站初期要做什麼。


【1. 公版 + 私版的訂單成立邏輯】

1.1 公版商品下單時，什麼時候算訂單正式成立？
   (A) 客戶送出就成立
   (B) 付款完成才成立
   (C) HJ 人工確認後才成立

1.2 私版客製商品（即時報價或 LINE 報價後）下單，什麼時候算訂單正式成立？
   (A) 客戶確認報價就成立
   (B) 付款完成才成立
   (C) HJ 人工確認後才成立

1.3 如果客戶把公版 + 私版商品放一起結帳，希望怎麼處理？
   (A) 同一張訂單，公版先出、私版製完再補出
   (B) 系統自動拆成兩張訂單，各自處理
   (C) 規定客戶要分開結帳


【2. 訂單管理】

2.1 HJ 是否需要在後台幫客人修改訂單？通常會改哪些內容（數量 / 收件 / 付款方式 / 商品 / 取消品項）？

2.2 訂單出貨狀態目前有哪些名稱？(例：待確認、已成立、備貨中、已出貨、已完成、已取消)

2.3 撿貨單怎麼匯出？
   說明：撿貨單是倉庫員工實際拿貨用的內部單據。當有多張訂單要出貨時，倉管不會一張一張看訂單去拿，而是把多張訂單的商品彙整列印一張單，照單一次去倉庫拿。
   想問 HJ 目前的做法：
   (A) 按商品彙總（例：今天總共要出 200 個紙杯）
   (B) 按訂單一張一張列
   (C) 按物流批次（黑貓一批、超商一批）
   (D) 還沒做 / 不需要

2.4 退換貨目前怎麼處理？網站初期只需要提供申請入口，還是要做完整退換貨流程？


【3. 金流（付款方式）】

3.1 網站正式上線時要支援哪些付款方式？需求表已列 4 類：
   - 一般匯款
   - 綠界（信用卡 / ATM / 超商代碼可選）
   - 宅配貨到付款
   - 自取貨到付款
   想確認：網站初期就做這 4 類，還是要增減？

3.2 如果支援綠界，要開哪些付款方式：信用卡、ATM、超商代碼，還是只開信用卡？

3.3 一般匯款付款後誰確認入帳？網站是否需要客戶上傳匯款資訊？

3.4 貨到付款只適用公版商品嗎？私版客製商品也可用嗎？

3.5 (補充題) HJ 是否需要月結客戶功能？
   說明：需求表沒列月結，這是我們補問的（B2B 常見做法）。如果 HJ 不做月結，這題可跳過。如果要做，請說明：
   - 哪些會員等級可以用月結？
   - 每筆月結訂單都要人工審核嗎？還是老客戶可自動通過？
   - 由誰審？業務 / 主管 / 會計？

3.6 客戶要求退款或取消訂單時，後續處理由誰負責？
   說明：一般退款 / 取消訂單會牽動下面 4 件事，每項都可能不同人處理：
   (a) 金流退款（信用卡退刷 / 匯款退回客戶帳戶 / 月結沖帳）
   (b) 發票作廢或開立折讓單
   (c) 庫存歸還
   (d) ERP 訂單反向處理
   想問 HJ 目前是怎麼分工的？


【4. 物流與出貨】

4.1 四大超商、宅配、自取，網站正式上線時是否都要做？

4.2 物流單目前是怎麼產生的？
   說明：物流單是給物流商的出貨託運單。HJ 目前的做法：
   (A) 已串接物流商系統，由物流商系統產生
   (B) HJ 自己用 Excel / 內部系統整理填寫
   (C) 還沒做 / 人工填單
   網站初期是否要做？怎麼做？

4.3 物流狀態（已出貨 / 配送中 / 已送達）希望怎麼更新到網站？
   說明：自動更新需要跟物流商 API 串接，會增加開發與維護成本。HJ 希望：
   (A) 串接物流商 API 自動更新（成本較高）
   (B) 不串接，HJ 出貨後在後台手動改狀態
   (C) 不做狀態追蹤，由物流商系統呈現給客戶查

4.4 自取訂單是否也要有付款與取貨狀態？

4.5 離島地區是否一律改由客服確認，不走網站一般配送？

4.6 運費怎麼算？
   - 依重量、體積、區域，還是訂單金額？
   - 合作客戶（A 級 / B 級）有特別運費優惠嗎？


【5. ERP 串接（凌越）】

5.1 你們目前跟凌越的串接方式是哪一種？
   (A) 即時 API（下完單馬上推 ERP）
   (B) 排程批次（每 N 分鐘 / 每天定時）
   (C) 業務手動匯入

5.2 訂單、顧客、庫存、商品資料，哪些一定要即時同步？哪些半自動或人工匯入即可？

5.3 ERP 匯入失敗時，網站訂單是否仍成立還是擋下單？

5.4 新網站會員如何對應原 ERP 客戶編號？

5.5 網站正式上線時若先用半自動或人工匯入，HJ 是否可接受？


【6. 庫存】

6.1 庫存數量是以 ERP 為準，還是網站也會自己維護？

6.2 庫存要在哪個階段檢查？(商品頁 / 加入購物車 / 結帳送出)

6.3 「預留庫存」適用哪些商品？加入購物車就預留，還是訂單成立後才預留？

6.4 缺貨提醒要提醒誰：客戶、HJ 業務、倉管，還是只在後台顯示？

6.5 私版商品需要庫存概念嗎？還是只做報價 / 接單？


【7. 發票】

7.1 企業客戶（三聯式 / 抬頭 / 統編）系統記住下次自動帶 OK 嗎？還是每次都要重新確認？

7.2 個人會員要強制綁手機條碼載具嗎？還是可以選擇只用 Email 寄電子發票？

3. 庫存是否以 ERP 為準？是否要即時同步？
```
