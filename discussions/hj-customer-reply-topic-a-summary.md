## 一句話結論

HJ 第一版應將公版商品、樣品、私版詢價分成三種清楚流程：公版走一般訂單，樣品走獨立樣品申請單，私版走詢價單 + LINE。前台可讓入口清楚銜接，但資料與流程不要混在同一張訂單裡。

---

## 客戶已回覆的事實

### 公版商品

- 公版商品有規格層級：尺寸、容量、顏色、材質、數量（箱 / 條）。
- 不同規格組合會有不同價格。
- 不同規格組合庫存分開計算。
- 凌越內每個規格組合都有自己的商品編號。

### 樣品

- 全部公版商品都可以申請樣品。
- 需要登入會員才可成功送出樣品申請。
- 每個品項最多 3 個。
- 最多 10 款品項，總共最多 30 個。
- 樣品免費，但需要收運費。
- 樣品不可與販售的公版商品一起出貨。

### 加購商品

- 加購商品可能出現在購物車 / 結帳頁推薦。
- 加購商品也可能是買特定商品時才出現指定加購品。
- 加購品價格也會依會員等級不同。

### 私版商品

- 私版商品細節、原始檔、確認下單都走 LINE。
- 網站不需要做檔案上傳、線上校稿、線上確認完稿。
- 網站先建立私版詢價單，再引導客戶加入官方 LINE。
- HJ 希望跳轉 LINE 時可以帶出詢價單編號。
- 雙方確認後才支付訂金、下訂成功並開始製作。

## 目前暫定流程

### 1. 公版商品購買流程

公版商品走一般商品購買流程。

流程暫定：

```text
公版商品列表 / 詳情
-> 選購既有商品或規格品項
-> 加入購物車
-> 結帳
-> 成立一般訂單
```

目前判斷：

- 「尺寸、容量、顏色、材質、數量（箱 / 條）」主要是在講公版商品資料。
- 這些規格資料不代表前台一定要做成同頁規格選擇器。
- 高機率是 HJ 已建立好不同規格組合，每個規格組合對應價格、庫存與凌越商品編號。
- 但前台到底要用「選現成品項」還是「同一商品頁切換規格」，需要再問 HJ。

### 2. 樣品申請流程

樣品不混入一般商品訂單，改走獨立樣品申請單。

流程暫定：

```text
客戶點「索取樣品」
-> 登入會員
-> 選樣品品項與數量
-> 系統檢查限制
-> 選配送方式
-> 計算樣品運費
-> 付款運費
-> 成立樣品申請單
```

目前判斷：

- 樣品本身免費，但網站流程中需要有「收運費」這一步。
- 樣品申請單應獨立成立，不與一般商品訂單混在一起。
- 樣品申請紀錄建議出現在會員中心，方便客戶查詢申請紀錄與配送狀態。
- 樣品運費算法、可用配送方式、狀態名稱先留到 B / F 討論。

### 3. 私版詢價流程

私版商品不走一般購物車，先走詢價單 + LINE / 人工確認。

流程暫定：

```text
私版商品選單 / 分類
-> 私版商品詳情
-> 填寫客製需求
-> 送出私版詢價單
-> 顯示詢價編號與 LINE 引導
-> HJ 人工確認檔案、價格、訂金與製作內容
-> 確認後才進付款與正式訂單階段
```

目前判斷：

- 私版不是一般購物車商品。
- 網站後台應保留私版詢價單，不能只留在 LINE 對話。
- LINE 是客服溝通與檔案傳遞渠道，不是正式資料庫。
- 訂金付款、發票、何時轉正式訂單，留到 F / E 討論。

### 4. 加購商品流程

加購品第一版先只綁公版商品的一般購買流程。

流程暫定：

```text
公版商品詳情 / 購物車 / 結帳
-> 顯示可加購商品
-> 加入一般購物車
-> 與公版商品一起成立一般訂單
```

目前判斷：

- 加購品不放入樣品申請流程。
- 加購品不放入私版詢價流程。
- 加購品與主商品的關聯規則、庫存、會員價、凌越品號，留到 D / E / C 再細談。

### 5. 公版與私版前台選單

公版與私版前台應分開選單 / 分開分類。

建議結構：

```text
公版商品
-> 公版商品列表
-> 公版商品詳情
-> 加入購物車 / 申請樣品 / 加購品

私版商品
-> 私版商品列表
-> 私版商品詳情
-> 客製需求 / 詢價
-> LINE / 人工確認
```

目前判斷：

- 不需要把私版詢價硬塞進公版商品頁。
- 可在導覽或適當位置提示「需要客製請至私版商品」，但不是公版商品的主流程。

## HJ 待確認問題（A 包暫存）

以下只是 A 包暫存題，還不是最後正式問卷。等 A-F 都收斂後，再統一潤成給 HJ 的版本。

1. 公版商品的尺寸、容量、顏色、材質、箱 / 條，是 HJ 先建立成不同商品 / 品項讓客戶選購，還是希望在同一商品頁內讓客戶自行切換規格後再加入購物車？
2. 樣品申請是否確認獨立於一般商品訂單，不與公版商品混在同一張訂單裡？
3. 樣品申請完成後，會員中心是否需要顯示樣品申請紀錄、狀態與物流追蹤連結？
4. 私版商品是否確認不直接加入一般購物車，而是先建立私版詢價單，再透過 LINE / 人工確認報價與製作內容？
5. 私版詢價單送出後，是否需要顯示詢價編號，並在 LINE 溝通時可對應該筆詢價？
6. 加購品是否只適用於公版商品的一般購買流程？
7. 加購品是由 HJ 後台設定「哪些主商品可搭配哪些加購品」，還是所有商品共用同一批加購品？

## 留到後面主題包

### 留到 B：物流 / 運費

- 樣品運費算法。
- 樣品可用配送方式。
- 樣品是否可超商 / 自取。
- 樣品配送狀態名稱。
- 加購品是否跟主商品一起計算運費。
- 公版商品箱購 / 條購 / 箱購 + 條購的材積與配送限制。

### 留到 F：金流 / 發票

- 樣品運費怎麼收款。
- 樣品運費是否開發票。
- 私版訂金怎麼付款。
- 私版訂金是否開發票。
- 私版尾款或正式訂單付款方式。

### 留到 C：會員 / 價格

- 加購品會員價怎麼套用。
- 樣品是否限制特定會員類型。
- 私版詢價是否開放給所有會員，或只開放企業 / 特定會員。

### 留到 D：商品後台 / 庫存

- 樣品是否扣庫存。
- 樣品可申請數量如何設定。
- 加購品是否有自己的庫存。
- 公版規格資料如何匯入 / 維護。
- 私版商品欄位是否與公版商品分開管理。

### 留到 E：訂單 / ERP / 後台作業

- 樣品是否進凌越。
- 樣品是否需要後台出貨流程。
- 私版詢價何時進凌越，是否先進報價單。
- 訂金支付後是否才轉正式訂單。
- 加購品在 ERP 訂單明細中如何表示。
