# HJ 商品規格與物流規劃觀察

> 建立日期：2026-05-03
> 用途：先觀察 HJ 現站商品規格與物流限制，判斷新網站物流規劃應如何落地。
> 狀態：初步觀察；不是完整全站商品盤點。

---

## 1. 目前觀察到的 HJ 現站商品規格型態

HJ 現站商品頁多半是一般商品描述文字，常見欄位包含：

- 品名
- 容量
- 尺寸
- 材質
- 單位
- 箱規
- 搭配
- 描述

不同商品欄位不完全一致。

### 觀察例子

| 商品類型 | 現站可見規格 |
|---|---|
| 塑膠壽司盒 | 尺寸、材質、單位、箱規 |
| 木質餐匙 | 尺寸、材質、單位、箱規 |
| 塑膠便當盒 | 容量、尺寸、材質、單位、箱規、搭配 |
| PLA 杯 | 容量、尺寸、材質、單位、箱規、搭配 |
| 耐熱袋 | 尺寸、材質、單位 |
| 垃圾袋 | 容量、尺寸、材質、單位 |
| 熱感紙捲 | 尺寸、材質、單位、箱規 |

### 初步判斷

現站已有很多商品「尺寸 / 單位 / 箱規」資料，但目前看起來比較像文字描述，不像已經被拆成可計算欄位。

物流判斷若要自動化，不能只靠這段文字，需要把資料拆成結構化欄位。

---

## 2. 現站商品資料對物流判斷的缺口

若新網站要做配送方式判斷，至少需要以下欄位：

| 欄位 | 用途 |
|---|---|
| 商品類型 | 判斷箱購 / 條購 / 樣品 / 私版 |
| 單品長寬高 | 判斷單品是否可能超過超商限制 |
| 箱規長寬高 | 判斷箱購是否可超商 / 宅配 |
| 重量 | 判斷是否超過超商或宅配限制 |
| 材積 | 計算 HJ 運費表 |
| 是否可超商 | 快速控制配送方式 |
| 是否只能宅配 | 快速控制配送方式 |
| 是否需客服確認 | 處理特殊品 / 離島 / 超規商品 |

目前客戶已回覆會自行填入商品「材積」，且需要填整箱及單條材積。這是第一版很重要的基礎。

## 3. HJ 回覆文件中的物流 / 商品規格重點

以下不是我們假設，是 HJ 已在回覆文件中提供的方向。

### 商品規格

- 公版商品規格層級包含：尺寸、容量、顏色、材質、數量（箱、條）。
- 不同規格組合的價格會不同。
- 不同規格組合的庫存會分開計算。
- 凌越裡每個規格組合都有自己的商品編號。

### 材積與運費

- HJ 會自行填入商品材積。
- 需要填入「整箱」與「單條」材積。
- 材積用途有兩個：
  - 判斷能不能超商配送。
  - 計算宅配運費。
- 若超過超商限制，網站應跳出只剩宅配可選。

### 超商限制

- 四大超商都要支援：7-11、全家、萊爾富、OK。
- 不能超商配送的原因是「不符合超商寄送規定」。
- 已查到的公開限制：
  - 7-ELEVEN 一般交貨便：最長邊 `<= 45cm`，長 + 寬 + 高 `<= 105cm`，重量 `<= 10kg`。
  - 全家常溫店到店：長 + 寬 + 高 `<= 105cm`，單邊 `<= 45cm`，重量 `<= 5kg`。
  - 萊爾富店到店：最長邊 `<= 45cm`，長 + 寬 + 高 `<= 105cm`，重量 `40g < 重量 <= 5kg`。
  - OK 標準常溫中尺寸：長 + 寬 + 高 `<= 105cm`，最長邊 `<= 45cm`，重量 `<= 5kg`；OK 官方另有 120cm / 10kg 大尺寸規格，但是否適用 HJ 電商串接要再確認。

備註：7-ELEVEN 目前公開資料已可見常溫一般交貨便 `10kg` 規範，全家官方店到店仍是 `5kg`。HJ 回覆若目前用 `45 x 30 x 30 cm / 5kg` 作為超商限制，提案階段需再確認這是 HJ 內部保守規則，還是尚未更新到各通路最新限制。

### 宅配與出貨

- 多家宅配是 HJ 出貨時填寫，不是客戶下單時選宅配公司。
- HJ 目前配合宅配：
  - 新航（配送最多）
  - 嘉里大榮
  - 新竹貨運
  - 黑貓
  - 大嘴鳥
- 客戶查貨態可以提供物流追蹤連結即可。
- 出貨目前偏人工處理。
- 新網站需要：
  - 宅配出貨：輸入配送編號，可單筆或批次。
  - 超商出貨：需要批次列印寄件單。

### 訂單 / 配送狀態

- 會員中心要看到：
  - 備貨中
  - 已出貨
  - 配送中
  - 已送達
  - 訂單已取消
- 狀態目前由管理部人員 / 業務更新。
- 訂單後台狀態也需要可篩選：
  - 待付款
  - 已付款
  - 備貨中
  - 已出貨
  - 已完成
  - 已取消

## 4. 物流平台限制的角色

物流平台會公布尺寸、重量、材積、離島、冷凍等規則。

但規則存在不代表物流平台會替網站完整判斷每一張購物車是否可寄。

新網站通常要自己負責：

- 讀商品尺寸 / 材積 / 重量
- 加總購物車商品
- 判斷可用配送方式
- 不符合超商時隱藏超商
- 不符合宅配或規則不明時改為客服確認

物流平台主要負責：

- 建立物流訂單
- 選門市
- 產生託運單
- 物流狀態通知
- 物流編號 / 貨態查詢

---

## 5. HJ 第一版建議不要做太重

不要一開始就規劃成完整物流決策系統。

第一版建議先做「基本配送方式判斷 + HJ 運費規則 + 人工出貨資訊」。

### 第一版建議做

1. 商品後台可填：
   - 箱購 / 條購
   - 單條材積
   - 整箱材積
   - 重量
   - 是否可超商
   - 是否只能宅配
2. 結帳時依規則顯示配送方式：
   - 箱購是否開放超商，先問 HJ 確認是否一定超過超商限制
   - 若部分箱購仍可能符合超商限制，就依整箱材積 / 重量判斷
   - 若 HJ 確認所有箱購都不適合超商，第一版可直接設定箱購只開放宅配 / 自取
   - 條購依材積 / 重量檢查是否可超商
   - 超過超商限制只顯示宅配
   - 超過可判斷範圍顯示聯絡客服
3. 依 HJ 運費表計算：
   - 宅配免運門檻
   - 超商免運門檻
   - 條購材積級距運費
4. 後台出貨時可填：
   - 宅配公司
   - 物流編號
   - 物流追蹤連結
5. 支援單筆或 Excel 批次更新物流資訊。

### 第一版先不要做

- 出貨時每筆都重新輸入實際包裹長寬高重量。
- 自動判斷每一家宅配公司的詳細可寄限制。
- 多家宅配 API 全部串接。
- 自動拆箱 / 拆包。
- 自動依物流商回傳完整貨態。

這些可列為第二階段或進階功能。

## 6. 中間方案：只串貨態，不串建單

這是 Wayne 2026-05-03 討論中確認的可行方向。

可以不讓網站負責建立物流單，仍維持 HJ 現況：

```text
客戶下單
→ 網站先做配送方式 / 運費基本判斷
→ HJ 後台審核訂單
→ HJ 到配合物流平台 / 物流商系統建單出貨
→ HJ 取得物流公司 + 物流編號
→ HJ 回填到網站後台
→ 網站用物流編號查貨態，或由物流平台回傳貨態
→ 會員中心顯示配送狀態與追蹤資訊
```

這種做法不是完整物流自動化，而是：

> 人工建單 + 網站同步貨態。

### 需要串接的範圍

只需串：

- 貨態查詢 API，或
- 貨態 webhook / 通知 API

不需要一開始串：

- 物流建單 API
- 自動選物流商
- 自動拆件
- 自動列印宅配單
- 建單失敗補償流程

### 仍需補問

1. HJ 出貨建單是否集中在同一個物流平台。
2. 若是同一平台，該平台是否提供貨態查詢 API 或貨態回傳。
3. 若是不同宅配公司各自建單，是否每一家都有可用貨態 API。
4. 網站狀態要顯示物流原始狀態，還是統一轉成 HJ 的狀態：備貨中、已出貨、配送中、已送達、訂單已取消。

### 初步判斷

如果 HJ 所有宅配都透過同一個平台出貨，第二階段可以優先考慮「只串該平台貨態」。

如果 HJ 是分別到新航、嘉里大榮、新竹貨運、黑貓、大嘴鳥各自系統處理，貨態同步就會變成多家 API 整合，成本與維護會明顯提高。

## 7. 建議執行方式

### Step 1：先做商品規格盤點表

從現站抽出代表商品，不需要一開始全站逐筆規格化。

建議每大類至少抽 3-5 個商品：

- 餐盒
- 紙杯 / 塑膠杯
- 紙袋 / 塑膠袋
- 餐具
- 紙捲 / 耗材
- 樣品商品
- 私版商品

盤點欄位：

| 欄位 | 說明 |
|---|---|
| 商品名稱 | 現站商品名 |
| 分類 | 商品大類 |
| 單位 | 條 / 箱 / 包 / 件 |
| 是否有箱規 | 是 / 否 |
| 是否有重量 | 是 / 否 |
| 是否可算材積 | 是 / 否 |
| 是否可能超商 | 是 / 否 / 不確定 |
| 是否只能宅配 | 是 / 否 / 不確定 |
| 需要 HJ 補資料 | 要補哪些欄位 |

### Step 2：先定配送判斷規則

先用簡單規則，不做過度自動化：

```text
箱購
→ 先問 HJ 是否一定超過超商限制
→ 若不一定超規，檢查整箱材積與重量
→ 符合限制才顯示超商
→ 若 HJ 確認全部箱購都超規，才直接只開宅配 / 自取

條購
→ 檢查材積與重量
→ 符合超商限制才顯示超商
→ 不符合則只顯示宅配 / 自取

樣品
→ 依 A 包討論結果決定是否獨立申請與運費

私版
→ 不走購物車，轉詢價 / LINE
```

### Step 3：再決定是否需要物流 API

如果 HJ 目前仍是人工出貨，第一版可以先不急著串多家宅配 API。

可先做：

- 後台填物流公司
- 後台填物流編號
- 產生追蹤連結
- Excel 批次匯入物流資訊

超商如果要做線上選店與建單，再評估是否接綠界物流。

### Step 4：把進階功能放第二階段

進階項目包含：

- 多家宅配 API 自動建單
- 出貨時實際包裹尺寸檢查
- 自動拆箱 / 分件
- 物流異常自動通知
- 完整貨態自動同步

## 8. 目前對 HJ 的建議方向

HJ 商品不像一般小商品電商，不能只做「選地址 / 選超商」就結束。

但 HJ 現站也不像已經有完整複雜物流判斷，因此新網站第一版不建議一次拉到完整物流自動化。

建議提案語氣：

> 新網站第一版先依商品材積、重量、箱購 / 條購屬性做基本配送方式判斷與運費計算。超商不符合時不顯示超商選項，宅配由 HJ 出貨時人工選擇物流公司並填入物流編號。多家宅配 API、自動拆件、出貨包裹尺寸檢查可列為第二階段進階功能。

---

## 9. 待補確認

1. HJ 現有商品是否已有完整整箱 / 單條材積資料表。
2. HJ 是否願意補重量欄位。
3. 商品「箱購 / 條購」是否可由凌越商品資料判斷，還是要網站後台維護。
4. 樣品是否獨立配送與獨立運費。
5. 箱購是否一定超過超商限制；若不是，HJ 是否能提供整箱材積 / 重量供網站判斷。
6. 宅配運費是否完全照 HJ 提供的材積表，還是有不同宅配公司例外。
7. 第一版是否要串綠界超商選店 / 建單，或先只做宅配與自取。
8. 若要同步貨態，HJ 是否集中在同一物流平台建單；若沒有，是否接受先人工更新狀態與追蹤連結。
9. 超商限制要採各通路官方差異規則，還是 HJ 內部統一採保守 `45 x 30 x 30 cm / 5kg` 規則。
