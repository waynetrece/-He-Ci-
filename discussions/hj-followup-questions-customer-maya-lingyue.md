# HJ 後續確認問題清單：客戶 / 馬亞 / 凌越

> 日期：2026-05-01
> 來源：窗口回覆文件 + 凌越 ERP API 文件判讀
> 用途：把後續問題分流，避免把客戶業務問題、馬亞技術規劃、凌越 API 補件混在一起。

---

## 1. 給 HJ 客戶再確認

這一區只問 HJ 需要決定的業務規則，不問 API 細節。

### A. 樣品與一般商品

1. 樣品運費金額是多少？是否固定？是否依宅配 / 超商 / 材積變動？
2. 如果客戶同時選「樣品 + 公版商品」，HJ 希望：
   - A. 直接禁止一起結帳。
   - B. 同一頁結帳，但系統自動拆成樣品申請 + 一般訂單。
   - C. 可以一起付款，但分開出貨。
3. 樣品是否需要進凌越，還是只留在網站後台管理？
4. 樣品是否要扣庫存或占用可申請數量？

### B. 私版詢價與 LINE

1. 私版詢價成立後，網站是否只保留詢價紀錄，不進凌越？
2. 訂金支付後，才建立正式訂單，這樣是否符合 HJ 作業？
3. LINE 跳轉時帶出詢價單編號，是希望：
   - A. 客戶看到可複製的詢價編號。
   - B. 跳到 LINE 時訊息框自動帶入詢價編號。
   - C. 客戶加入 LINE 後由客服手動查詢。
4. 私版詢價狀態需要讓會員在會員中心查詢嗎？

### C. 商品、價格與會員

1. HJ 的會員/客戶價格類別總共有幾種？是否只會落在凌越內建的 5 種價格？
2. 北部直客、中南部直客、北部盤商、中南部盤商之外，是否還有其他價格類別？
3. 舊客戶資料是否都有 Email、手機或統編，可用來對應新網站會員？
4. 新會員補上 ERP 客戶編號前，是否只能看新網站訂單，不能看舊訂單？

### D. 訂單與狀態

1. 未付款訂單是否也要立即進凌越？
2. 訂單同步凌越失敗時，HJ 希望：
   - A. 直接不讓客戶完成下單。
   - B. 網站先成立訂單，標示同步失敗，由管理員補送。
   - C. 網站自動重送，超過次數再通知管理員。
3. 「已完成」目前說 7 天後自動跳轉，起算點是已出貨日、物流到貨日，還是人工改為已出貨後第 7 天？
4. 客戶是否可以自己取消未付款訂單？還是所有取消都由管理員操作？

### E. 物流、材積與運費

1. 商品材積中的「箱」與「條」如何在商品規格上判斷？
2. 運費圖中的「才」是否就是 HJ 會填入的材積單位？
3. 箱購 + 條購同時下單時，是否依附圖「箱購 + 條購」規則計算？
4. 超商尺寸限制 45 x 30 x 30 cm，是看單一商品、單一箱、還是整張訂單包裹後尺寸？
5. 超商 5 公斤限制，是看單一商品還是整張訂單總重量？
6. 自取訂單是否需要「可取貨」狀態？目前只回覆收到訂單/備貨中，但客戶會需要知道何時可取。

### F. 金流、退款與發票

1. 綠界 ATM 虛擬帳號是否確定要做？目前回覆是「考慮」。
2. HJ 綠界帳號由誰申請，預計何時可提供測試與正式資料？
3. 匯款截圖由誰審核？會計還是管理部人員？
4. 發票由哪裡開立：凌越、綠界，還是網站另接電子發票服務？
5. 退貨由客服後台協助申請時，是否需要在會員中心顯示退貨/退款狀態？

## 2. 給馬亞技術評估

這一區是請馬亞判斷系統怎麼做比較穩，不是直接丟給客戶。

### A. 整體架構

1. 網站是否應保留自己的訂單資料庫，再同步到凌越，而不是每次都只依賴凌越回應？
2. 訂單同步是否需要佇列、重送、失敗通知與人工補送頁？
3. 網站訂單狀態應由網站維護，還是由凌越回寫？
4. 凌越金鑰 30 秒失效，程式端應如何包裝 token / pass key 取得流程？

### B. 商品與價格

1. 凌越商品資料的 5 組價格是否足以支援 HJ 的多地區 / 多客戶類型價格？
2. 如果價格類型超過 5 種，網站是否需要自己的價格表？
3. Shopline Excel 匯入資料是否先進網站，再轉凌越，還是只用於新網站初始化？
4. SEO、商品文案、網站上下架屬於網站功能，應由網站獨立管理，不列入給凌越的 API 補件問題；商品圖片若要從 ERP 帶出，只確認圖片檔名 / 取檔方式。

### C. 庫存

1. 凌越目前庫存 API 若沒有 webhook，網站要用即時查詢、排程同步，還是結帳前二次確認？
2. 預留給連鎖店的庫存，要在網站做可售量規則，還是從凌越可用量直接取得？
3. 缺貨提醒可否用網站通知系統處理，而不是依賴凌越？
4. LINE 缺貨提醒是通知 HJ 內部人員，還是通知客戶？這兩種技術做法不同。

### D. 樣品與購物車

1. 樣品是否獨立成 sample_request 資料，不進一般 order？
2. 客戶同時加入樣品與公版商品時，前台是否同頁展示但送出後拆成兩筆流程？
3. 樣品運費是否能套用一般運費計算器，還是獨立規則？

### E. 物流與材積

1. HJ 的運費圖規則是否可以做成後台可維護規則表？
2. 箱購 / 條購 / 箱購+條購的材積加總公式要如何落在資料結構中？
3. 四大超商寄件單是否應另接物流 API，例如綠界物流或其他物流整合服務？
4. 宅配物流編號單筆 / 批次匯入要用 Excel，還是做後台批次表單？

### F. 金流與退款

1. 綠界信用卡與 ATM callback 如何更新網站訂單狀態？
2. 付款成功後是否才同步凌越，還是下單未付款就先同步？
3. 信用卡退刷成功後，是否要同步凌越銷退單 / 收款沖銷 / 發票作廢？
4. 匯款付款審核是否需要後台人工審核畫面？

### G. LINE

1. LINE 跳轉時自動帶詢價單編號，可否用官方支援的方式完成？
2. 若要推送詢價摘要給客戶，是否需要 LINE Login 或 account linking 取得 line userId？
3. 若只做「跳轉 LINE + 帶詢價編號」，是否可先不做 Messaging API 推播？

## 3. 給凌越確認 / 補資料

這一區應請 HJ 轉給凌越或由馬亞直接與凌越確認。

### A. API 環境與權限

1. 請提供 HJ 專案實際 Web Service endpoint / WSDL。
2. 請提供測試環境 URL。
3. 請提供測試帳號、密碼、公司代號。
4. API 主機是否需固定 IP 白名單？若需要，白名單流程為何？
5. HJ 目前授權可使用哪些資料種類 idakd？
6. 各 API 是否有呼叫頻率限制或資料筆數限制？

### B. 商品資料

1. 商品規格組合是否每一筆都以 `SK_NO` 作為唯一商品編號？
2. HJ 的箱 / 條單位，應對應商品基本資料還是單位換算設定檔？
3. 商品材積、重量、整箱材積、單條材積應使用哪些欄位？
4. 商品圖片若由凌越提供，API 回傳的是檔名、路徑還是可下載檔案？請提供取檔方式。
5. 會員價格超過 5 種時，凌越是否有客戶別商品價格表 API？

### C. 客戶與會員

1. 客戶資料是否可依 Email、手機、統編查詢？
2. 新網站會員建立後，是否可由 API 新增凌越客戶？
3. 若新網站先建立會員、之後才補 ERP 客戶編號，凌越建議流程為何？
4. 客戶價格類型、地區別、客戶類別、業務指定等欄位，HJ 實際使用哪幾個？

### D. 訂單同步

1. 網站訂單要轉入凌越時，建議使用 `0000A0 訂單資料` 還是其他單據？
2. 訂單編號由網站產生或凌越產生？
3. API 轉入成功後回傳的主鍵或單號格式為何？
4. 未付款訂單是否可先轉入凌越？
5. 訂單明細如果有加購品、樣品、運費、折扣，應如何表示？
6. 訂單同步失敗時，哪些錯誤可重送，哪些錯誤需人工處理？
7. 是否有範例 XML 可提供：一般商品訂單、含發票、含運費、含不同會員價。

### E. 庫存同步

1. 目前庫存 `000009` 是否只能由網站查詢，還是有庫存異動推播？
2. `@WD_AMT` 與 `@WD_AMTOK` 哪個應作為網站可販售庫存？
3. 預留給連鎖店的庫存是否已反映在可用量？
4. 是否支援依多倉庫查詢可用量？
5. 庫存資料建議多久同步一次？

### F. 歷史訂單

1. 會員中心歷史訂單應讀訂單資料、銷貨單，還是兩者都讀？
2. 若讀銷貨單，是否可回推原訂單、商品明細、物流資訊？
3. 是否可依客戶編號查詢歷史訂單並分頁？
4. 舊訂單是否有物流公司、物流編號、付款狀態欄位？

### G. 報價、退貨、收款與發票

1. 私版詢價是否建議轉入 `0000A5 客戶報價單`？
2. 客戶支付訂金後，報價單如何轉正式訂單？
3. 信用卡付款或 ATM 付款完成後，是否要建立凌越收款單？
4. 退貨退款應使用銷退單、收款沖銷，還是其他流程？
5. 發票如果由凌越開立，電子發票欄位與流程請提供 HJ 實際範例。

### H. 物流與出貨

1. 凌越 API 是否有欄位可存宅配公司與物流編號？
2. 凌越是否支援批次匯入物流編號？
3. 凌越是否支援超商寄件單產生？若不支援，是否建議另接物流服務？
4. 撿貨單 API 文件清單有列出，但完整欄位規格請補提供。
5. 出貨狀態是否可由 API 更新或匯出？

## 4. 優先順序

### 第一優先：不確認會卡架構

1. 訂單同步凌越的實作方式與失敗處理。
2. 庫存即時更新可行性。
3. 會員分級價資料來源。
4. 樣品與公版商品分流。
5. 材積 / 運費 / 超商限制計算。
6. 綠界付款與退款責任分工。

### 第二優先：不確認會影響畫面

1. 私版詢價成功頁與 LINE 帶編號。
2. 會員歷史訂單顯示範圍。
3. 自取可取貨狀態。
4. 缺貨提醒是否用 LINE。
5. 訂單狀態給客戶看的文字。

### 第三優先：可先合理預設

1. 商品預約上 / 下架畫面。
2. 商品匯入錯誤報告格式。
3. 宅配物流編號單筆 / 批次匯入 UI。
4. 撿貨單按商品 + 物流方式分組。
