## 1. 重讀後的一句話結論

這份 207 頁 API 文件可以證明凌越 ERP 具備商品、客戶、訂單、銷貨、銷退、收款、報價、發票、庫存、倉庫、入出庫等資料匯出入能力，但它仍是「凌越通用資料結構文件」，還不是 HJ 專案可直接開工的完整串接規格。

馬亞可以依這份文件開始設計整合架構，但仍需要凌越補 HJ 實際環境、權限、欄位對照、XML 範例、狀態規則與測試資料；也需要 HJ 再確認幾個會影響畫面與流程的業務規則。

## 3. 對 HJ 系統架構的重新判斷

### 3.1 網站仍應保留自己的主資料與流程狀態

API 文件有 ERP 單據欄位，但網站不應只把凌越當成即時資料庫使用。原因：

1. API 金鑰 30 秒會失效，呼叫需要包裝與重試。
2. 文件沒有看到交易型保證、webhook、同步佇列、明確 SLA。
3. HJ 的前台狀態，例如待付款、已付款、備貨中、配送中、已送達、已完成、已取消，不一定能直接對應凌越欄位。
4. 樣品申請、私版詢價、LINE 跳轉、SEO、預約上下架、商品頁文案等都是網站領域，不列入給凌越的 API 補件問題。

建議架構：

- 網站保留商品展示資料、會員資料、購物車、樣品申請、私版詢價、訂單狀態、金流 callback、物流追蹤資料。
- 凌越作為 ERP 主系統，負責商品 / 客戶 / 訂單 / 庫存 / 銷貨 / 收款 / 發票等資料同步。
- 串接層要有同步紀錄、失敗重送、錯誤訊息、人工補送功能。

### 3.2 商品資料不能只看凌越商品基本資料

凌越商品 `000000` 有 SKU、品名、規格、單位、輔助單位、價格 1~5、重量、體積、每箱材積、材積單位、圖片檔名等欄位。這足以支援許多 ERP 商品資料。

但 HJ 網站還需要：

- 前台分類與排序。
- 商品 SEO。
- 商品頁長文、圖片排版、使用情境。
- 預約上架 / 下架。
- 加購規則。
- 樣品可申請設定。
- 私版 / 公版商品分流。
- 商品是否顯示於網站。

這些不應拿去問凌越是否支援。比較穩的做法是：ERP 商品編號作為 SKU 核心，網站另外管理前台展示欄位；問凌越只問 ERP 商品欄位、價格、單位、材積重量、圖片檔名或取檔方式。

### 3.3 價格規則是目前最需要釐清的核心之一

API 中商品有 `SK_PRICE1~5`、`SK_FPRICE1~5`，客戶有 `CT_PAYFG` 售價種類，客戶類別有 `000007`。這代表凌越至少支援 5 組基本價與客戶售價類型。

HJ 回覆提到會員價依地區 / 客戶類型，例如北部直客、中南部直客、北部盤商、中南部盤商，且每個商品各自設定不同等級價格。若實際價格類型超過 5 種，或存在「客戶別商品價」，`000000` 的 5 組價可能不夠。

所以不能直接承諾「凌越商品價全部可直接套網站會員價」。必須請凌越確認是否另有客戶別價格、客戶貨品項號對照或專案價 API。

### 3.4 庫存可以查，但「即時」要重新定義

`000009 目前庫存` 可轉出目前庫存 `@WD_AMT` 與可用量 `@WD_AMTOK`，也有倉庫、批號、有效日資訊。這對商品頁、結帳前庫存檢查很有用。

但 207 頁中沒有看到庫存異動 webhook / push 事件。若 HJ 說「凌越一改網站馬上更新」，技術上要先拆成三種可能：

1. 前台顯示時即時向凌越查詢。
2. 網站高頻排程同步凌越庫存。
3. 網站保留快取，但結帳前再查凌越可用量。

若要真正秒級即時推播，需要凌越補其他機制；目前文件不能證明有。

### 3.5 訂單進凌越可行，但要先決定未付款與失敗處理

`0000A0 訂單` 有完整抬頭與明細欄位，可放客戶、地址、付款方式、發票、商品、價格、數量、材積、重量、備註等。這是網站公版訂單進凌越的主要候選。

但仍需確認：

- 未付款訂單是否先進凌越。
- 網站訂單編號與凌越單號誰主導。
- 訂單同步失敗時，前台是否讓客戶完成下單。
- 運費、折扣、加購品、樣品是否作為明細或備用欄。
- 訂單狀態由網站維護，還是凌越回寫。

### 3.6 私版詢價可對應客戶報價單，但不應先假設一定要進 ERP

`0000A5 客戶報價單` 結構與訂單相似，理論上可對應私版詢價。可是 HJ 回覆明確說私版細節、原始檔、確認下單都走 LINE，網站不做上傳與校稿。

因此應先問 HJ：

- 網站成立詢價單時就要同步凌越報價單嗎？
- 還是 LINE 確認金額後，才由業務 / 網站送 ERP 報價？
- 訂金付款後才建立正式訂單嗎？

若沒先決定，後續會影響私版成功頁、會員中心詢價紀錄、LINE 帶單號、訂金支付與 ERP 報價 / 訂單轉換。

### 3.7 物流與超商標籤不是這份 API 能直接保證的範圍

API 有商品重量、材積、訂單 / 銷貨 / 出庫資料，也有運送方式類型可用代碼管理。但沒有看到四大超商寄件單、超商標籤列印、宅配託運單產生、物流狀態 callback 的完整 API。

所以 HJ 的物流需求要拆開：

- 運費與可配送方式：網站依 HJ 規則計算。
- 宅配公司與追蹤碼：網站後台 / 匯入功能處理。
- 四大超商寄件單批次列印：大概率要另接物流 / 金物流服務，不應算在凌越 API 已涵蓋。

補充：若 HJ 所說的「金物流」是綠界 ECPay，金物流可負責建立物流訂單、列印託運單、查詢物流訂單與接收貨態通知，但材積 / 重量是否可用超商或宅配，仍應由網站先依 HJ 規則判斷。不要假設金物流 API 會替網站完整判斷每一張購物車是否可超商配送。

獨立補充文件：`discussions/hj-ecpay-logistics-volume-judgement.md`

### 3.8 金流、退款與電子發票要分責任

凌越 API 有收款單、銷貨退回、銷項發票欄位，能處理 ERP 端記錄。但綠界信用卡、ATM 虛擬帳號、付款 callback、退刷 API 並不在凌越文件裡。

因此架構上要分清楚：

- 綠界負責付款 / 退刷。
- 網站負責接 callback、更新訂單付款狀態、記錄付款資料。
- 凌越負責收款 / 銷退 / 發票資料同步，前提是 HJ 決定由凌越開發票或記帳。

## 4. API 文件目前足夠與不足

這一段的判斷標準是：**馬亞能不能只看這份文件就直接寫 API 對接程式**。不是拿 API 文件去問 SEO、前台文案、網站上下架這些跟凌越無關的網站功能。

目前結論：

- 文件對「凌越有哪些 API 函式、有哪些資料類別、欄位名稱是什麼」說明清楚。
- 文件對「HJ 這個專案要怎麼實際對接、每個流程 XML 怎麼填、哪些欄位必填、測試環境怎麼連」還不夠。
- 所以馬亞可以用這份文件開始設計對接架構，但還需要凌越補 HJ 專案實際對接資料，才能正式開發與測試。

### 4.1 足夠的部分

這份文件足以讓馬亞知道：

- 凌越採 SOAP Web Service。
- 主要函式是 `LyGetPassKey`、`LyDataIn`、`LyDataOut`、`LyDataDel`、`LyDataPage`。
- 金鑰 30 秒失效。
- XML 格式以 `DocumentElement`、`LYDATATITLE`、`LYDATADETAIL` 為主。
- 可依 `idakd` 操作各種資料類別。
- 商品、客戶、訂單、報價、收款、銷貨、銷退、發票、庫存、倉庫等欄位可查。
- 大部分單據都有新增、覆蓋、重編單號、權限檢查等 `imode` 參數。

### 4.2 不足以直接開發的部分

仍缺 HJ 專案實作必需資料：

1. 實際 endpoint / WSDL。
2. 測試與正式環境。
3. 帳號、密碼、公司代號。
4. 固定 IP / 防火牆 / TLS 要求。
5. HJ 已授權可用的 `idakd` 清單。
6. 每個流程的實際 XML 範例。
7. HJ 欄位對照表。
8. 必填欄位與預設值。
9. 單號規則。
10. API 錯誤碼、可重送條件、頻率限制。
11. 庫存同步建議。
12. 訂單 / 銷貨 / 出庫 / 物流狀態對照。
13. 發票與金流實際責任切分。
14. 客戶別價格或專案價資料來源。
15. 超商寄件單 / 撿貨單完整 API 或替代方案。

## 5. 會影響製作的核心防漏功能

| 功能 | 必須先確認的原因 | 目前判斷 |
|---|---|---|
| 樣品流程 | 會影響購物車、結帳、運費、訂單、出貨 | 應獨立於一般訂單，是否進 ERP 待確認 |
| 私版詢價 | 會影響 LINE、報價單、訂金、會員中心 | 可對應 `0000A5`，但進 ERP 時機待確認 |
| 會員價格 | 會影響商品價格資料結構 | 5 組價格可能不夠，需確認客戶別價格 |
| 商品資料來源 | 會影響後台、匯入、SEO、上架排程 | ERP 管 SKU，網站管前台展示較穩 |
| 訂單同步 | 會影響下單能否成立與後台補送 | 可用 `0000A0`，但失敗處理需設計 |
| 庫存同步 | 會影響超賣與前台庫存可信度 | `000009` 可查，真正即時未證明 |
| 運費 / 材積 | 會影響結帳金額與超商可配送 | API 有材積欄位，但運費規則是網站邏輯 |
| 超商標籤 | 會影響出貨後台工時 | 凌越文件未證明支援，可能需另接物流 |
| 金流 / 退款 | 會影響訂單付款狀態與退刷 | 綠界另接，凌越只處理 ERP 收款 / 銷退 |
| 發票 | 會影響 B2B/B2C、載具、捐贈 | 凌越欄位有，但開立責任需確認 |
| 歷史訂單 | 會影響會員中心 | 要決定讀訂單、銷貨單，或兩者 |
| 出貨狀態 | 會影響客戶查詢 | ERP 欄位不等於前台狀態，網站需自管或映射 |
